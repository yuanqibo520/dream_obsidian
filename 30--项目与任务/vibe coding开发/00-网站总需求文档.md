# 大学校内交谈网站需求文档

## 1. 产品定位

面向大学生的校内交流社区，先以“本校信息流 + 匿名表达 + 评论互动 + 基础治理”验证用户活跃度，再逐步扩展到校园服务、二手交易、活动与学习互助等模块。

一期不做校园超级平台，优先保证发帖、浏览、评论、消息回流和内容治理闭环可用。

## 2. 目标用户

- 本校在校大学生：关注校内热点、吐槽、提问、求助、社团和活动信息。
- 校园内容维护人员：负责板块维护、内容审核、举报处理和违规用户处置。
- 后续扩展用户：校友、商家、校方运营人员，暂不作为一期核心用户。

## 3. 核心目标

- 验证学生是否愿意持续发布和评论校内内容。
- 验证匿名发帖是否提升表达意愿和活跃度。
- 建立本校内容池，让用户打开首页即可看到有价值的校内信息。
- 通过评论、点赞、@ 提及和系统通知带动用户回流。
- 建立基础审核和举报机制，控制匿名社区的内容风险。

## 4. 一期功能范围（P0 必做）

### 4.1 账号与基础身份

- 支持手机号注册、登录、退出登录。
- 支持选择学校，进入对应学校社区。
- 支持编辑昵称、头像、年级、专业、个人简介。
- 预留学生认证字段，首版不强制接入学号或教务系统。
- 支持账号设置、隐私设置和注销申请入口。

### 4.2 社区板块

- 默认板块：校园新鲜事、吐槽灌水、学术交流、社团活动、失物招领、表白墙、校园公告。
- 板块支持名称、简介、排序、启用 / 停用状态。
- 管理员可创建、编辑、隐藏或删除板块。
- 校园公告板块建议仅管理员或认证运营人员可发布。

### 4.3 发帖与内容发布

- 支持发布标题、正文、图片。
- 支持选择板块后发布。
- 支持匿名发布，匿名身份仅在当前帖子内展示，不影响后台追责。
- 支持编辑和删除自己发布的帖子。
- 支持帖子置顶、加精，由管理员操作。
- 支持草稿保存，降低发帖中断导致的内容丢失。

一期暂不支持视频、语音、投票、付费内容和复杂富文本。

### 4.4 浏览、搜索与排序

- 首页展示本校帖子信息流。
- 支持热门 / 最新排序。
- 支持按板块浏览。
- 支持关键词搜索帖子和板块。
- 支持帖子详情页查看正文、图片、评论和楼中楼回复。
- 热门排序一期可先使用点赞数、评论数、发布时间的简单权重，不做个性化推荐算法。

### 4.5 评论与互动

- 支持帖子评论。
- 支持楼中楼回复，建议限制嵌套层级，避免讨论结构过深。
- 支持点赞帖子和评论。
- 支持收藏帖子。
- 支持 @ 用户并触发通知。
- 支持举报帖子和评论。

### 4.6 消息中心

- 评论回复通知。
- 点赞通知。
- @ 提及通知。
- 收藏或精华等内容状态变更通知。
- 系统通知。
- 支持消息已读 / 未读状态。

### 4.7 个人中心

- 展示用户基础资料。
- 查看我的帖子、评论、收藏。
- 支持头像、昵称、年级、专业、简介编辑。
- 支持隐私设置、消息免打扰、黑名单。
- 支持反馈与投诉入口。

### 4.8 管理与治理

- 管理员可审核、删除、隐藏帖子和评论。
- 支持举报列表、举报分类、处理状态和处理结果记录。
- 支持敏感词过滤和发布前提示。
- 支持用户禁言、封禁、解封和违规记录查看。
- 支持管理员查看基础数据：新增用户、发帖量、评论量、举报量、热门板块。
- 后台保留匿名帖真实用户映射，用于违规追责，前台不展示。

## 5. 一期暂不做（避免范围失控）

- 私信 / 私聊、好友、关注流、同校匹配、动态广场。
- 教务课表同步、一卡通、校园导航、跑腿代取等校方或线下服务。
- 二手交易的担保支付、交易评价、在线议价闭环。
- 兼职招聘、Offer 爆料、职业规划内容库。
- 学习资料付费、课程评分体系、学习打卡、考研资料库。
- 多主题皮肤、复杂积分等级、徽章体系。
- 复杂推荐算法、跨校推荐、AI 图片 / 视频审核。
- 商业化广告、会员、校园商家入驻。

## 6. 后续扩展方向（P1 / P2）

### 6.1 P1：社区增强

- 关注用户和关注流。
- 活动发布与报名。
- 更完善的新手引导和帮助中心。
- 举报处理进度对用户可见。
- 基础积分或成长值，用于鼓励高质量发帖和评论。
- 用户标签、话题标签和板块订阅。

### 6.2 P2：校园服务与交易

- 二手交易板块，先支持信息发布和线下自提，不直接做担保支付。
- 失物招领增强，包括物品分类、地点、联系方式保护。
- 课程评价、选课经验和学习资料分享。
- 校园地图、课表提醒、一卡通等能力需在校方接口可用后再评估。
- 校友身份和校友互动。

## 7. 内容安全与隐私要求

- 匿名只影响前台展示，不代表不可追责。
- 用户发布内容需遵守社区规范，禁止人身攻击、隐私曝光、诈骗、色情、暴力、政治敏感和违法内容。
- 举报需包含类型、对象、理由、处理状态和处理结果。
- 敏感内容可先进入待审核状态。
- 用户可拉黑其他用户，减少骚扰。
- 用户可申请注销账号和删除个人数据。
- 后台操作需记录管理员、操作对象、操作时间和操作原因。

## 8. 非功能需求

- 移动端优先，兼容主流手机浏览器。
- 首页信息流加载速度应稳定，支持分页或下拉加载。
- 发布、评论、点赞等核心操作需要有明确成功 / 失败反馈。
- 图片上传需限制数量、大小和格式。
- 搜索、发帖、评论、举报等高频接口需要基础防刷限制。
- 系统需支持基础日志，便于排查登录、发布、审核和消息问题。

## 9. 核心流程

### 9.1 新用户首次使用

1. 注册或登录。
2. 选择学校。
3. 完善基础资料。
4. 浏览本校热门内容。
5. 发布帖子或参与评论。
6. 收到互动消息后回流。

### 9.2 发帖流程

1. 点击发帖入口。
2. 选择板块。
3. 输入标题、正文并上传图片。
4. 选择是否匿名。
5. 发布成功后进入帖子详情页。

### 9.3 举报处理流程

1. 用户举报帖子或评论。
2. 系统记录举报类型和理由。
3. 管理员在后台处理。
4. 处理结果写入记录。
5. 必要时对内容或用户执行删除、隐藏、禁言、封禁。

## 10. 验收标准

- 用户可完成注册、登录、退出登录。
- 用户可选择学校并进入本校首页。
- 用户可发布普通帖和匿名帖。
- 用户可浏览首页、板块页和帖子详情页。
- 用户可评论、楼中楼回复、点赞、收藏、@ 用户。
- 用户可搜索帖子和板块。
- 用户可收到评论、点赞、@ 提及和系统通知。
- 用户可举报违规帖子或评论。
- 管理员可管理板块、审核内容、处理举报、封禁用户。
- 匿名帖前台隐藏真实身份，后台可追溯发布者。

## 12. 成功指标建议

- 注册转化率。
- 完善学校信息比例。
- 日活跃用户数。
- 人均浏览帖子数。
- 发帖率和评论率。
- 匿名帖占比。
- 消息点击回流率。
- 举报处理时长。
- 被删除或被举报内容占比。
