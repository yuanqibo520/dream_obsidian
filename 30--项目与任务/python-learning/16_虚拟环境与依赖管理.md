# 虚拟环境与依赖管理

## 为什么需要虚拟环境

不同项目可能依赖不同版本的库。虚拟环境可以隔离每个项目的依赖。

## 创建虚拟环境

```bash
python -m venv .venv
```

## 激活虚拟环境

Windows PowerShell：

```bash
.venv\Scripts\Activate.ps1
```

macOS/Linux：

```bash
source .venv/bin/activate
```

## 安装依赖

```bash
pip install requests
```

## 导出依赖

```bash
pip freeze > requirements.txt
```

## 安装依赖文件

```bash
pip install -r requirements.txt
```

## 常见工具

- `pip`
- `venv`
- `pip-tools`
- `poetry`
- `uv`

## 练习

1. 创建一个虚拟环境。
2. 安装 `requests` 并写一个请求网页的脚本。
3. 导出 `requirements.txt`。
