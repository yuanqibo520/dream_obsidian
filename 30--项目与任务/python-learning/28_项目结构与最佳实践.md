# 项目结构与最佳实践

## 推荐结构

```text
my_project/
  README.md
  requirements.txt
  src/
    my_project/
      __init__.py
      main.py
  tests/
    test_main.py
```

## 代码风格

- 使用清晰的变量名。
- 函数只做一件事。
- 避免超长函数。
- 避免重复代码。
- 用类型注解表达意图。

## 格式化工具

常见工具：

- black
- ruff
- isort

安装：

```bash
pip install black ruff isort
```

## README 内容

一个好 README 通常包含：

- 项目简介
- 安装方式
- 运行方式
- 配置说明
- 测试方式
- 示例

## 配置文件

常见配置方式：

- `.env`
- `config.toml`
- `config.yaml`
- 环境变量

## 错误处理建议

- 不要吞掉异常。
- 给用户清晰的错误信息。
- 给开发者保留日志。

## 练习

1. 为一个小项目创建标准目录结构。
2. 添加 README。
3. 添加测试目录。
4. 使用 black 格式化代码。
