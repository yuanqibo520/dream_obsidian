# 项目实战路线

## 入门项目

- 命令行计算器。
- 猜数字游戏。
- 文本文件统计器。
- 学生成绩管理系统。

## 进阶项目

- 简单日志库。
- JSON 配置读取器。
- 迷你命令行 Todo 工具。
- 使用 STL 实现常见数据结构练习。

## 工程项目

- CMake 多模块项目。
- 带单元测试的小型工具库。
- 简单 HTTP 客户端或服务端。
- 2D 小游戏或图形窗口程序。

## 每个项目都练什么

- 需求拆分。
- 目录结构。
- 接口设计。
- 错误处理。
- 测试和调试。
- README 文档。

## 推荐节奏

每学完一个模块，就做一个小练习。不要等“全部学完”再开始写代码，C++ 必须在实践中建立手感。
