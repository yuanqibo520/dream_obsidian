# 动态内存管理

## 栈与堆

局部变量通常位于栈上，离开作用域会自动销毁。

```cpp
void f() {
    int x = 10;
}
```

动态申请的对象通常位于堆上，需要明确的所有权管理。

```cpp
int* p = new int(10);
delete p;
```

## new 和 delete

`new` 创建对象并返回指针，`delete` 销毁对象并释放内存。

```cpp
User* user = new User("Alice");
delete user;
```

数组形式必须成对使用：

```cpp
int* nums = new int[10];
delete[] nums;
```

不要混用 `new[]` 和 `delete`，也不要混用 `new` 和 `delete[]`。

## 常见内存问题

- 内存泄漏：申请后没有释放。
- 重复释放：同一块内存释放两次。
- 悬空指针：指针指向已经释放或销毁的对象。
- 越界访问：访问数组或缓冲区范围之外的内存。

## 所有权

所有权回答一个问题：谁负责释放资源？

```cpp
auto user = std::make_unique<User>("Alice");
```

`std::unique_ptr` 明确表达独占所有权，离开作用域时自动释放资源。

## RAII

现代 C++ 不鼓励在业务代码里直接手写 `new` 和 `delete`。更推荐用对象生命周期管理资源。

```cpp
{
    std::vector<int> nums;
    nums.push_back(1);
}
```

`nums` 离开作用域时自动释放内部动态内存。

## 实践建议

- 默认使用自动对象和标准库容器。
- 需要动态对象时优先使用 `std::make_unique`。
- 确实需要共享所有权时使用 `std::make_shared`。
- 裸指针更多用于“不拥有对象”的观察关系。
- 写 C++ 时要同时思考对象生命周期和资源所有权。
