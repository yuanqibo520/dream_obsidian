# 结构体、枚举、联合

## struct

```cpp
struct Point {
    double x;
    double y;
};
```

`struct` 适合表达简单数据聚合。

## enum class

枚举适合表示“一组有限的命名取值”。例如颜色、方向、菜单选项、程序状态、错误码等，都可以用枚举表达。

```cpp
enum class Color {
    Red,
    Green,
    Blue
};
```

优先使用 `enum class`，它比传统 `enum` 更安全：

- 枚举值有自己的作用域，使用时要写 `Color::Red`。
- 不会自动转换成 `int`，可以避免把枚举值误当成数字使用。
- 不同枚举类型之间不会随便混用。

```cpp
Color c = Color::Red;

if (c == Color::Red) {
    // ...
}
```

### 枚举的本质

枚举值背后仍然是整数。默认从 `0` 开始，后面的值依次加 `1`。

```cpp
enum class Direction {
    Up,     // 0
    Down,   // 1
    Left,   // 2
    Right   // 3
};
```

也可以手动指定枚举值：

```cpp
enum class ErrorCode {
    Ok = 0,
    NotFound = 404,
    ServerError = 500
};
```

### 指定底层类型

默认情况下，枚举的底层整数类型由编译器决定。需要控制大小或二进制格式时，可以显式指定。

```cpp
#include <cstdint>

enum class ByteFlag : std::uint8_t {
    Read = 1,
    Write = 2,
    Execute = 4
};
```

这里 `ByteFlag` 的底层类型是 `std::uint8_t`，通常占 1 个字节。

### 和 switch 搭配

枚举经常和 `switch` 搭配，用来处理有限状态。

```cpp
enum class GameState {
    Menu,
    Playing,
    Paused,
    GameOver
};

void update(GameState state) {
    switch (state) {
    case GameState::Menu:
        // 显示菜单
        break;
    case GameState::Playing:
        // 更新游戏逻辑
        break;
    case GameState::Paused:
        // 暂停
        break;
    case GameState::GameOver:
        // 结算
        break;
    }
}
```

如果以后新增了枚举值，编译器可能会提醒 `switch` 没有处理完整，这比用一堆整数或字符串判断更可靠。

### enum class 和传统 enum

传统 `enum` 的枚举值会进入外层作用域，而且容易自动转换成整数。

```cpp
enum ColorOld {
    Red,
    Green,
    Blue
};

int x = Red; // 可以自动转换成 int
```

这会带来两个问题：

- `Red`、`Green`、`Blue` 这些名字可能和其它变量或枚举冲突。
- 枚举值可以被当成普通整数传来传去，类型约束较弱。

`enum class` 解决了这些问题：

```cpp
enum class Color {
    Red,
    Green,
    Blue
};

Color c = Color::Red;
// int x = c; // 错误：不能自动转换成 int
```

如果确实需要转成整数，可以显式转换：

```cpp
int value = static_cast<int>(Color::Red);
```

### 什么时候用枚举

适合用枚举：

- 一组固定选项：`Red`、`Green`、`Blue`。
- 有限状态：`Idle`、`Running`、`Stopped`。
- 明确的类别：`Student`、`Teacher`、`Admin`。


### 小例子

```cpp
#include <iostream>

enum class Role {
    Guest,
    User,
    Admin
};

bool canDelete(Role role) {
    return role == Role::Admin;
}

int main() {
    Role role = Role::User;

    if (canDelete(role)) {
        std::cout << "可以删除\n";
    } else {
        std::cout << "权限不足\n";
    }

    return 0;
}
```

这个例子里，`Role` 比直接使用 `int` 更清晰。看到 `Role::Admin`，就能知道它表示一种角色，而不是一个随意的数字。

## union

```cpp
union Value {
    int i;
    double d;
};
```

`union` 的多个成员共用同一块内存。普通业务代码很少直接使用，现代 C++ 更常用 `std::variant`。

## 建议

- 简单数据用 `struct`。
- 有限状态用 `enum class`。
- 多类型值优先考虑 `std::variant`。
