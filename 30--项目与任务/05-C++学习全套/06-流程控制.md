## range-for

`range-for` 又叫范围 `for` 循环，适合用来遍历数组、字符串、`vector`、`list`、`map` 等“可以被遍历”的对象。

基本语法：

```cpp
for (元素声明 : 被遍历的对象) {
    循环体
}
```

例如：

```cpp
for (const auto& item : items) {
    std::cout << item << '\n';
}
```

它的含义是：每次从 `items` 中取出一个元素，临时命名为 `item`，然后执行循环体。

### 遍历数组

```cpp
int nums[] = {1, 2, 3, 4, 5};

for (int n : nums) {
    std::cout << n << '\n';
}
```

这里的 `n` 会依次得到 `1`、`2`、`3`、`4`、`5`。

### 遍历 vector

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> nums = {10, 20, 30};

    for (int n : nums) {
        std::cout << n << '\n';
    }
}
```

当不需要下标，只需要逐个访问元素时，`range-for` 比普通 `for` 更简洁。

### 拷贝遍历

```cpp
for (auto item : items) {
    std::cout << item << '\n';
}
```

这种写法会把容器中的每个元素复制一份给 `item`。

优点是简单、安全；缺点是如果元素很大，复制成本可能比较高。

适合遍历 `int`、`char`、`double` 这类小对象：

```cpp
std::vector<int> nums = {1, 2, 3};

for (auto n : nums) {
    std::cout << n << '\n';
}
```

### 引用遍历

如果想修改容器里的元素，要使用引用：

```cpp
std::vector<int> nums = {1, 2, 3};

for (auto& n : nums) {
    n *= 2;
}

// nums 变成 {2, 4, 6}
```

如果写成下面这样，就只能修改副本，原容器不会改变：

```cpp
for (auto n : nums) {
    n *= 2;
}
```

### const 引用遍历

如果只读取元素，不修改元素，推荐使用 `const auto&`：

```cpp
for (const auto& item : items) {
    std::cout << item << '\n';
}
```

这种写法有两个好处：

1. `const` 表示不会修改元素。
2. `&` 表示不复制元素，效率更高。

所以遍历字符串、结构体、类对象等较大的元素时，经常写成：

```cpp
for (const auto& s : names) {
    std::cout << s << '\n';
}
```

### 遍历字符串

`std::string` 也可以用 `range-for` 遍历：

```cpp
#include <iostream>
#include <string>

int main() {
    std::string text = "hello";

    for (char ch : text) {
        std::cout << ch << '\n';
    }
}
```

如果想修改字符串中的字符：

```cpp
#include <cctype>

std::string text = "hello";

for (char& ch : text) {
    ch = std::toupper(ch);
}

std::cout << text << '\n'; // HELLO
```

### 遍历 map*

`map` 中的每个元素是一个键值对：

```cpp
#include <iostream>
#include <map>
#include <string>

int main() {
    std::map<std::string, int> scores = {
        {"Alice", 90},
        {"Bob", 85}
    };

    for (const auto& pair : scores) {
        std::cout << pair.first << ": " << pair.second << '\n';
    }
}
```

其中：

- `pair.first` 是键。
- `pair.second` 是值。

C++17 以后可以使用结构化绑定，让写法更清楚：

```cpp
for (const auto& [name, score] : scores) {
    std::cout << name << ": " << score << '\n';
}
```

### 什么时候不用 range-for

`range-for` 适合“从头到尾遍历全部元素”。如果需要下标，就用普通 `for` 更合适：

```cpp
for (std::size_t i = 0; i < nums.size(); ++i) {
    std::cout << i << ": " << nums[i] << '\n';
}
```

需要从后往前遍历时，也通常使用普通 `for`：

```cpp
for (int i = static_cast<int>(nums.size()) - 1; i >= 0; --i) {
    std::cout << nums[i] << '\n';
}
```

遍历容器时，如果不需要下标，优先考虑 `range-for`。只读遍历较大的对象时，优先使用 `const auto&`。


## 扩展
### C++17和C++20的区别
C++17、C++20 后面的数字基本指的是 **C++ 标准发布/定稿的年份**

- C++14：2014 年
- C++17：2017 年
- C++20：2020 年
- C++23：2023 年标准，正式 ISO 文档在 2024 年发布
- C++26：2026 年标准，WG21 在 2026 年 3 月 28 日完成/批准进入最终流程

截至今天，也就是 **2026 年 5 月 31 日**，最新的是 **C++26**。不过实际项目里，编译器、标准库、第三方库完全支持 C++26 还需要时间，所以生产项目不会立刻大规模切过去。WG21 官网也显示 2026 年 4 月、5 月已经有 post-Croydon / pre-Brno 邮件列表，说明 C++26 后标准工作继续推进中。[![](https://www9.open-std.org/favicon.ico)WG21](https://www9.open-std.org/JTC1/SC22/WG21/index.html)

目前最多人使用的版本大概率还是 **C++17**。JetBrains 2025 C++ 调查里，C++17 是最常用标准，占 **43%**；C++20 是 **34%**；C++23 是 **21%**；C++26 只有 **7%** 早期采用者

### vector库函数存储三个以上的变量怎么遍历

##### 方法：用 tuple
```C++
#include <iostream>
#include <vector>
#include <tuple>
using namespace std;

int main() {
    vector<tuple<int, string, double>> arr;

    arr.push_back({1, "Tom", 90.5});
    arr.push_back({2, "Jack", 88.0});

    for (auto v : arr) {
        cout << get<0>(v) << " "
             << get<1>(v) << " "
             << get<2>(v) << endl;
    }

    return 0;
}
```
输出：
```C++
1 Tom 90.5
2 Jack 88
```

### 引用和指针的区别是什么？
```C++
int a = 10;
int b = 20;

int& r = a;   // r 是 a 的引用，也就是 a 的别名
int* p = &a;  // p 是指针，保存 a 的地址

r = 30;       // 修改的是 a
*p = 40;      // 也是修改 a

p = &b;       // 指针可以改为指向 b
// r = b;     // 不是让 r 改为引用 b，而是把 b 的值赋给 a
```
