# 构造函数与析构函数

## 构造函数

构造函数负责初始化对象。

```cpp
class User {
public:
    // 构造函数：创建 User 对象时必须提供 name 和 age。
    // name 按值传入，调用者可以传左值，也可以传临时字符串。
    User(std::string name, int age)
    // 成员初始化列表：在对象真正构造时直接初始化成员。
    // std::move(name) 把参数 name 中的字符串资源移动给 name_，避免一次不必要的拷贝。
    // age_(age) 用参数 age 初始化成员变量 age_。
    : name_(std::move(name)), age_(age) {}

private:
    // 成员变量通常加下划线，用来和构造函数参数区分。
    std::string name_;
    int age_;
};
```

优先使用成员初始化列表。它不是“先默认构造再赋值”，而是在对象创建时直接初始化成员，通常更高效，也能初始化 `const` 成员和引用成员。

## 析构函数

析构函数负责对象销毁时的清理。

```cpp
class File {
public:
    ~File() {
        close();
    }
};
```

## 拷贝控制

拷贝控制指的是：一个对象被拷贝、赋值、移动、销毁时，类应该如何处理自己的内部资源。

如果类管理资源，需要考虑这五个特殊成员函数：

- 拷贝构造函数。
- 拷贝赋值运算符。
- 移动构造函数。
- 移动赋值运算符。
- 析构函数。

### 三法则、五法则、零法则

如果一个类需要自己写析构函数，通常说明它在手动管理资源，例如裸指针、文件句柄、网络连接等。这时也要认真考虑拷贝和赋值，否则容易出现浅拷贝、重复释放、资源泄漏等问题。

- 三法则：如果需要手写析构函数、拷贝构造函数、拷贝赋值运算符中的一个，通常也需要考虑另外两个。
- 五法则：C++11 加入移动语义后，还要考虑移动构造函数和移动赋值运算符。
- 零法则：优先使用 `std::string`、`std::vector`、`std::unique_ptr`、`std::shared_ptr` 等标准库类型管理资源，让编译器自动生成这些特殊成员函数。

### 什么时候需要手写

像 `User` 这种类只包含 `std::string` 和 `int`，一般不需要手写拷贝控制函数：

```cpp
class User {
public:
    User(std::string name, int age)
    : name_(std::move(name)), age_(age) {}

private:
    std::string name_;
    int age_;
};
```

`std::string` 自己已经正确处理内存拷贝、移动和释放，`int` 也不需要特殊处理。所以让编译器自动生成拷贝构造、拷贝赋值、移动构造、移动赋值和析构函数就可以。

真正需要手写的常见情况是类直接管理资源：

```cpp
class Buffer {
public:
    Buffer(std::size_t size)
    : size_(size), data_(new int[size]) {}

    ~Buffer() {
        delete[] data_;
    }

private:
    std::size_t size_;
    int* data_;
};
```

这个类如果使用编译器默认拷贝，会让两个 `Buffer` 指向同一块数组。两个对象析构时会对同一块内存 `delete[]` 两次，程序就会出问题。

解决方式通常有三种：

- 禁止拷贝：把拷贝构造和拷贝赋值写成 `= delete`。
- 实现深拷贝：拷贝对象时重新分配一块新内存，再复制数据。
- 使用标准库类型：把 `int*` 换成 `std::vector<int>` 或智能指针，优先走零法则。

### 拷贝构造函数

用已有对象创建新对象时调用。

```cpp
User a("Tom", 18);
User b = a;      // 调用拷贝构造函数
User c(a);       // 也是调用拷贝构造函数
```

典型形式：

```cpp
class User {
public:
    User(const User& other)
    : name_(other.name_), age_(other.age_) {}

private:
    std::string name_;
    int age_;
};
```

`const User& other` 表示“只读地引用另一个 User 对象”。用引用可以避免复制参数本身，用 `const` 表示不会修改被拷贝的对象。

### 拷贝赋值运算符

两个对象都已经存在，把一个对象的内容赋给另一个对象时调用。

```cpp
User a("Tom", 18);
User b("Jerry", 20);

b = a;           // 调用拷贝赋值运算符
```

典型形式：

```cpp
class User {
public:
    User& operator=(const User& other) {
        if (this == &other) {
            return *this;     // 防止自己给自己赋值
        }

        name_ = other.name_;
        age_ = other.age_;
        return *this;         // 返回当前对象，支持连续赋值：a = b = c
    }

private:
    std::string name_;
    int age_;
};
```

拷贝构造是“用旧对象创建新对象”，拷贝赋值是“两个对象都已存在，把右边对象的值复制到左边对象”。

### 移动构造函数

用一个即将不用的对象创建新对象时调用。移动不是复制资源，而是把资源的所有权转移过来。

```cpp
User makeUser() {
    return User("Tom", 18);
}

User u = makeUser();           // 可能调用移动构造，也可能被编译器直接优化掉
```

典型形式：

```cpp
class User {
public:
    User(User& &other) noexcept
    : name_(std::move(other.name_)), age_(other.age_) {}

private:
    std::string name_;
    int age_;
};
```

`User&&` 是右值引用，通常表示这个对象可以被“搬走”。移动函数一般写 `noexcept`，这样标准库容器在扩容、搬迁元素时更愿意使用移动操作。

移动构造函数的优点：

- 减少昂贵的深拷贝：对于字符串、动态数组、容器等资源，移动通常只需要转移内部指针或句柄，不需要重新分配内存并逐个复制元素。。
- 让资源所有权转移更清晰：移动语义表达的是“这个对象的资源可以交给另一个对象”，比隐式拷贝更符合资源管理的真实意图。
- 配合标准库容器性能更好：例如 `std::vector` 扩容时，如果元素支持 `noexcept` 移动构造，容器可以优先移动元素，而不是拷贝元素。

简单理解：拷贝像“复制一份文件”，移动像“把文件夹改个归属”。复制可能很慢，移动通常只改资源的拥有者，所以更快。

### 移动赋值运算符

两个对象都已经存在，把一个临时对象或即将不用的对象的资源转移给当前对象时调用。

```cpp
User a("Tom", 18);
User b("Jerry", 20);

b = std::move(a);              // 调用移动赋值运算符
```

典型形式：

```cpp
class User {
public:
    User& operator=(User& &other) noexcept {
        if (this == &other) {
            return *this;
        }

        name_ = std::move(other.name_);
        age_ = other.age_;
        return *this;
    }

private:
    std::string name_;
    int age_;
};
```

被移动后的对象仍然是有效对象，但它的具体内容不要依赖。比如 `std::string` 被移动后还能析构、还能重新赋值，但原来的字符串内容可能已经没有了。

### 析构函数

对象生命周期结束时自动调用，用来释放对象管理的资源。

```cpp
class File {
public:
    ~File() {
        close();
    }
};
```

析构函数不接收参数，也没有返回值。局部对象离开作用域、`delete` 一个动态对象、容器销毁元素时，都会触发析构。



## 小结

普通类优先使用标准库类型管理资源，让编译器自动生成拷贝、移动和析构函数。只有当**类直接管理资源**时，才需要认真设计拷贝控制。
