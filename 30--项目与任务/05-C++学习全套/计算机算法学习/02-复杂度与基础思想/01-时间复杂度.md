# 时间复杂度

时间复杂度描述算法运行时间随输入规模增长的趋势，通常使用大 O 表示法。

## 常见复杂度

- `O(1)`：常数时间，例如数组按下标访问。
- `O(log n)`：对数时间，例如二分查找。
- `O(n)`：线性时间，例如遍历数组。
- `O(n log n)`：常见于高效排序，如归并排序、快速排序平均情况。
- `O(n^2)`：双重循环，例如朴素枚举两数。
- `O(2^n)`：指数级，例如枚举所有子集。
- `O(n!)`：阶乘级，例如全排列暴力枚举。

## 分析方法

只保留增长最快的项，忽略常数系数。

例如：

```text
3n^2 + 10n + 100 => O(n^2)
```

## 常见判断

- 单层循环通常是 `O(n)`。
- 两层独立嵌套循环通常是 `O(n^2)`。
- 每次规模减半通常是 `O(log n)`。
- 分治中每层处理 `n` 个元素且层数 `log n`，通常是 `O(n log n)`。

## 易错点

- 不要只数循环层数，要看变量如何变化。
- 哈希表平均 `O(1)`，最坏可能退化。
- 递归要考虑每层代价和递归树深度。
