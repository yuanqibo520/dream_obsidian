# 函数

## 函数定义

```cpp
int add(int a, int b) {
    return a + b;
}
```

## 函数声明

```cpp
int add(int a, int b);
```

声明告诉编译器函数存在，定义提供具体实现。

## 参数传递

```cpp
void by_value(int x);
void by_reference(int& x);
void by_const_reference(const std::string& s);
```

建议：

- 小对象可以按值传递。
- 大对象优先用 `const T&`。
- 需要修改实参时用 `T&`。

## 函数重载

```cpp
int max_value(int a, int b);
double max_value(double a, double b);
```

函数重载就是：同一个作用域里，可以有多个同名函数，只要它们的**参数列表**不同
具体包括：
- 类型不同
- 个数不同
- 顺序不同

注意：**返回值类型不同**不能构成重载

### const 对重载的影响

对于普通值参数，下面不能重载：
```C++
void f(int x); 
void f(const int x); 
// 错误，本质上是同一个函数
```

因为 int x 和 const int x 对调用者来说没区别，都是传一个 int 进去。

但是对于引用或指针，const 可以参与区分：

```C++
void f(int& x); 
void f(const int& x);
```

这两个可以重载。

例如：
```C++
int a = 10; 
const int b = 20; 
f(a); // 更倾向调用 int& 版本 
f(b); // 调用 const int& 版本 
f(3);// 调用 const int& 版本
```

因为字面量 3 不能绑定到普通 int&，但可以绑定到 const int&。

### 编译器如何选择重载函数

编译器会找“最匹配”的那个。

例如：
```C++
void show(int x); 
void show(double x); 
show(10); // int 版本，精确匹配 
show(3.14); // double 版本，精确匹配 
show('A'); // char 可以转换成 int，通常调用 int 版本
```


如果没有精确匹配，编译器会尝试类型转换。

但如果两个候选都差不多，就会报错。
```C++
void f(int x);
void f(double x); 
f(3.5f); 
// float 可以转 int，也可以转 double，但转 double 更自然，所以调用 double
```


再看一个可能出问题的例子：

```C++
void f(long x); 
void f(double x);
f(10); // 可能产生二义性
```

10 是 int，既可以转成 long，也可以转成 double。如果编译器无法判断哪个更好，就会报二义性错误。

## 默认参数

```cpp
void print(std::string text, int times = 1);
```

如果函数有声明和定义，默认参数一般写在声明里：
```C++
void print(std::string text, int times = 1);

void print(std::string text, int times) {
    for (int i = 0; i < times; i++) {
        std::cout << text << std::endl;
    }
}
```
注意：**默认参数必须从右往左提供**
