# 作用域与生命周期

## 作用域

作用域决定名字在哪里可见。

```cpp
int global_value = 1;

void f() {
    int local_value = 2;
}
```

`global_value` 是全局作用域，`local_value` 是局部作用域。

如果**局部变量和全局变量同名**，才需要区分。例如：
```C++
int value = 1; // 全局变量 

void f() {
int value = 2; // 局部变量 

cout << value; // 使用局部变量，输出 2 

cout << ::value; // 使用全局变量，输出 1 
}
```

关键点：

- 在函数内部，优先使用最近作用域里的变量，也就是局部变量。
- 如果局部变量和全局变量同名，局部变量会“遮蔽”全局变量。
- 想访问被遮蔽的全局变量，用 ::变量名。

## 生命周期

生命周期决定对象从何时创建到何时销毁。

```cpp
void f() {
    std::string name = "cpp";
}
```

`name` 在进入函数时创建，离开函数时销毁。

## static 局部变量

```cpp
int next_id() {
    static int id = 0;
    return ++id;
}
```

`static` 局部变量只初始化一次，生命周期持续到程序结束。
static 局部变量只是“活得久”，不是“看得见的地方变多”。  
主文件不能直接调用它，只能通过包含它的函数或返回引用的函数间接访问。

## 建议

尽量缩小变量作用域，让变量靠近使用位置。
