# 指针

## 核心概念

指针保存对象的地址。

```cpp
int x = 10;
int* p = &x;
```

`&x` 取得地址，`p` 保存地址。

## 解引用

```cpp
*p = 20;
```

`*p` 表示访问指针指向的对象。

## 空指针

```cpp
int* p = nullptr;
```

现代 C++ 使用 `nullptr`，不要使用 `NULL`。

## 指针风险

指针本质上只是一个地址，编译器无法自动保证这个地址一定有效，所以指针的主要风险都来自“指向哪里、还能不能用、该不该释放”。

### 1. 空指针解引用

空指针表示“不指向任何有效对象”。如果直接解引用，程序可能崩溃，也可能产生未定义行为。

```cpp
int* p = nullptr;
*p = 10; // 错误：不能解引用空指针
```

使用前要先判断：

```cpp
if (p != nullptr) {
    *p = 10;
}
```

### 2. 悬空指针

悬空指针指向的对象已经不存在了，但指针变量里还保存着原来的地址。

```cpp
int* p = nullptr;
{
    int x = 10;
    p = &x;
}

// x 已经离开作用域，p 变成悬空指针
// cout << *p; // 错误：未定义行为
```

常见来源：

- 指向局部变量，但局部变量已经离开作用域。
- `delete` 释放了堆内存，但指针还继续使用。
- 容器扩容后，之前保存的元素地址失效。

### 3. 内存泄漏

使用 `new` 申请内存后，如果没有对应的 `delete`，这块内存就无法再被正常释放。

```cpp
int* p = new int(10);
// 忘记 delete p; 会导致内存泄漏
```

如果中途发生 `return` 或异常，也很容易忘记释放资源。因此现代 C++ 更推荐使用智能指针或容器管理资源：

```cpp
#include <memory>

auto p = std::make_unique<int>(10);
```

### 4. 重复释放

同一块内存不能释放两次，否则会破坏堆内存结构。

```cpp
int* p = new int(10);
delete p;
delete p; // 错误：重复释放
```

如果必须手动释放，释放后可以把指针置空，降低误用概率：

```cpp
delete p;
p = nullptr;
```

### 5. 越界访问

指针可以做偏移运算，但它不会自动检查是否越过数组边界。

```cpp
int arr[3] = {1, 2, 3};
int* p = arr;

cout << *(p + 3); // 错误：越界访问
```

越界访问可能读到无关数据，也可能破坏其他内存，属于非常危险的未定义行为。

### 6. 类型不匹配

把一种类型的地址强行当成另一种类型使用，可能导致读取方式错误。

```cpp
double d = 3.14;
int* p = reinterpret_cast<int*>(&d); // 危险：把 double 的内存当 int 读
```

除非非常清楚底层内存布局，否则不要随意强制转换指针类型。

### 小结

指针风险可以归纳成三句话：

- 指针可能没有指向有效对象。
- 指针指向的对象可能已经失效。
- 指针可能越过它本该访问的范围。

## 建议

业务代码中优先使用引用、容器和智能指针，少直接管理裸指针。

- 能用引用表达“必须存在”的对象时，优先用引用。
- 能用 `std::vector`、`std::string` 管理连续数据时，不手写动态数组。
- 需要独占资源时，用 `std::unique_ptr`。
- 需要共享资源时，用 `std::shared_ptr`，但要注意循环引用问题。
- 如果只是临时观察对象，不负责释放，可以使用裸指针，但要明确它不拥有资源。
