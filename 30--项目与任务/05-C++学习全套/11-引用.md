# 引用

## 左值引用

引用是对象的别名。

```cpp
int x = 10;
int& r = x;
r = 20;
```

修改 `r` 等同于修改 `x`。

## const 引用

```cpp
void print(const std::string& text) {
    std::cout << text << '\n';
}
```

`const T&` 避免拷贝，同时保证函数不修改对象。

## 引用与指针区别

- **引用必须初始化。
- **引用通常不能重新绑定到另一个对象。**
- 指针可以为空，引用一般不表示空。

## 典型用途

- 函数参数避免拷贝。

  如果函数参数是普通值传递，会拷贝一份对象。对于 `int`、`double` 这种小类型，拷贝成本很低；但对于 `std::string`、`std::vector`、自定义类对象，拷贝可能比较贵。

  ```cpp
  void print(std::string text) {        // 会拷贝一份字符串
      std::cout << text << '\n';
  }

  void print(const std::string& text) { // 不拷贝，只绑定到原对象
      std::cout << text << '\n';
  }
  ```

  如果函数只是读取参数，通常使用 `const T&`：

  ```cpp
  void show(const std::vector<int>& nums) {
      for (int n : nums) {
          std::cout << n << ' ';
      }
  }
  ```

  如果函数需要修改传进来的对象，可以使用 `T&`：

  ```cpp
  void addOne(int& x) {
      ++x;
  }

  int a = 10;
  addOne(a); // a 变成 11
  ```

- 函数返回已有对象。

  函数可以返回引用，表示返回的不是一个新对象，而是某个已经存在的对象本身。

  ```cpp
  int& getFirst(std::vector<int>& nums) {
      return nums[0];
  }

  std::vector<int> v{1, 2, 3};
  getFirst(v) = 100; // 等价于 v[0] = 100
  ```

  这种写法常用于“允许调用者修改内部某个对象”的场景。

  但是要注意：不能返回局部变量的引用，因为局部变量在函数结束后就被销毁了。

  ```cpp
  int& bad() {
      int x = 10;
      return x; // 错误：x 离开函数后不存在
  }
  ```

  可以返回引用的对象，必须在函数结束后仍然存在，比如：

  - 函数参数引用。
  - 类的成员变量。
  - 静态变量。
  - 容器中仍然有效的元素。

- 实现运算符重载。

  运算符重载经常使用引用，主要是为了避免拷贝、支持链式调用，以及让语义更自然。

  例如重载加法时，两个参数通常使用 `const T&`，因为只需要读取它们：

  ```cpp
  class Point {
  public:
      int x;
      int y;
  };

  Point operator+(const Point& a, const Point& b) {
      return {a.x + b.x, a.y + b.y};
  }
  ```

  重载赋值运算符时，通常返回 `T&`，这样可以支持链式赋值：

  ```cpp
  class Person {
  public:
      std::string name;

      Person& operator=(const Person& other) {
          name = other.name;
          return *this;
      }
  };
  ```

  返回 `*this` 的引用后，就可以写：

  ```cpp
  a = b = c;
  ```

  其中 `b = c` 返回 `b` 本身，再赋值给 `a`。
