# 向量数据库

## 什么是向量数据库

向量数据库用于存储 Embedding，并支持根据向量相似度进行快速检索。

## 存储内容

一个向量记录通常包括：

- 向量本身。
- Chunk 文本。
- 文档 ID。
- 来源路径。
- 标题。
- 页码或章节。
- 权限标签。
- 更新时间。

## 常见向量数据库

| 名称 | 特点 |
|---|---|
| FAISS | 本地高性能向量检索库 |
| Milvus | 开源、适合大规模向量检索 |
| Qdrant | 开源、API 友好、过滤能力强 |
| Weaviate | 支持向量和对象管理 |
| Chroma | 轻量，适合原型和本地项目 |
| Elasticsearch | 可结合关键词检索和向量检索 |
| PostgreSQL pgvector | 适合已有 PostgreSQL 技术栈 |

## 索引

向量数据库通过索引提高检索速度。

常见索引类型：

- Flat：精确搜索，慢但准确。
- HNSW：近似搜索，速度快，常用。
- IVF：将向量聚类后搜索。

## Metadata Filter

元数据过滤非常重要。

例子：

- 只检索某个项目。
- 只检索用户有权限访问的文档。
- 只检索最近一年更新的资料。

## 选择建议

原型项目可用 Chroma 或 FAISS；生产系统可考虑 Qdrant、Milvus、Elasticsearch、pgvector 等方案。
