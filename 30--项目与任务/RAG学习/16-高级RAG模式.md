# 高级 RAG 模式

## Agentic RAG

Agentic RAG 让模型不只是一次检索，而是可以自主决定是否继续检索、换关键词、调用工具或验证答案。

适合复杂问题，但更难控制成本和稳定性。

## Graph RAG

Graph RAG 会构建实体、关系和社区结构，让系统能够基于知识图谱进行检索和总结。

适合：

- 复杂组织知识。
- 人物、项目、事件关系。
- 跨文档推理。

## Multi-Hop RAG

Multi-Hop RAG 用于需要多步推理的问题。

例子：

1. 找到某项目负责人。
2. 再查这个负责人相关的会议纪要。
3. 最后总结决策原因。

## Self-RAG

Self-RAG 让模型判断自己是否需要检索、检索结果是否足够、答案是否可信。

## Corrective RAG

Corrective RAG 会在检索结果质量差时进行纠正，例如重新检索、改写问题或扩大搜索范围。

## 多模态 RAG

多模态 RAG 可以处理文本、图片、表格、音频、视频等资料。

常见做法：

- 图片生成描述后入库。
- 表格转换成结构化文本。
- 音视频转写成文本。
- 保留原始媒体引用。

## 选择建议

不要一开始就做高级 RAG。先把基础 RAG 的数据、检索、引用和评估做好，再根据真实问题引入高级模式。
