# 上下文组装与 Prompt

## 上下文组装是什么

上下文组装是把检索到的 Chunk、用户问题、系统指令和输出格式要求组合成 Prompt。

## Prompt 基本结构

一个常见结构：

```text
你是一个知识库助手。
请只根据给定资料回答问题。
如果资料不足，请明确说不知道。

资料：
[1] ...
[2] ...
[3] ...

用户问题：
...

回答要求：
- 给出简洁答案。
- 标注引用编号。
- 不要编造资料外的信息。
```

## 上下文排序

通常把最相关的 Chunk 放在前面。对于长问题，也可以按逻辑顺序组织：

1. 定义和背景。
2. 规则和约束。
3. 操作步骤。
4. 示例。

## 上下文压缩

如果检索结果太长，需要压缩。

方法包括：

- 只保留与问题相关的句子。
- 删除重复内容。
- 合并同一文档的相邻 Chunk。
- 对候选内容先摘要再生成。

## 防止幻觉

Prompt 中应明确要求：

- 只基于上下文回答。
- 找不到答案就说资料不足。
- 不要使用常识补全关键事实。
- 输出引用来源。

## 好 Prompt 的标准

- 指令清晰。
- 资料边界明确。
- 输出格式稳定。
- 对无法回答的情况有约束。
