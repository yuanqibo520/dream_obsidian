# 工程实现流程

## 最小可用版本

一个最小 RAG 项目可以包含：

1. 文档上传。
2. 文档解析。
3. Chunk 切分。
4. Embedding 生成。
5. 向量存储。
6. 用户提问。
7. 检索相关 Chunk。
8. 调用大模型生成答案。
9. 返回引用。

## 数据入库流程

```text
文件 -> 解析 -> 清洗 -> 切分 -> 向量化 -> 写入数据库
```

需要记录：

- 文件 ID。
- Chunk ID。
- 文本内容。
- 向量。
- 元数据。
- 版本号。

## 问答流程

```text
问题 -> 问题改写 -> 检索 -> 重排序 -> 组装 Prompt -> 生成答案 -> 返回引用
```

## API 设计示例

| 接口 | 作用 |
|---|---|
| POST /documents | 上传文档 |
| POST /documents/index | 创建索引 |
| POST /chat | 提问 |
| GET /documents/{id} | 查看文档 |
| DELETE /documents/{id} | 删除文档及向量 |

## 日志记录

需要记录：

- 用户问题。
- 检索到的 Chunk。
- 相似度分数。
- Rerank 分数。
- 最终 Prompt 长度。
- 模型输出。
- 用户反馈。

## 工程注意事项

- 文档更新后要同步更新向量。
- 删除文档时要删除对应 Chunk。
- 检索必须受权限控制。
- 线上系统需要超时、重试和降级方案。
