# 答案生成与引用

## 答案生成目标

RAG 的答案应同时满足：

- 准确。
- 基于资料。
- 易读。
- 可追溯。
- 不编造。

## 引用来源

引用可以帮助用户判断答案是否可信。

常见引用方式：

- 文档标题。
- 文件路径。
- URL。
- 页码。
- Chunk 编号。
- 章节标题。

## 答案格式

根据场景选择格式：

| 场景 | 推荐格式 |
|---|---|
| 简单问答 | 直接回答加引用 |
| 流程说明 | 步骤列表 |
| 对比问题 | 表格 |
| 决策建议 | 结论、依据、风险 |
| 技术解释 | 概念、示例、注意事项 |

## 资料不足时

不要强行回答。可以输出：

```text
根据当前资料，无法确认该问题的答案。已有资料只提到了……，没有说明……
```

## 常见错误

- 检索结果里没有答案，但模型凭经验回答。
- 引用的资料和答案不对应。
- 把多个文档里的信息混合后产生错误结论。
- 忽略时间版本，使用过期资料。

## 改进方法

- 要求模型逐条引用。
- 对关键事实做二次验证。
- 显示“资料不足”的兜底答案。
- 对过期文档降低权重。
