# 数据准备与清洗

## 数据来源

常见数据包括：

- Markdown 笔记
- PDF 文件
- Word 文档
- 网页
- 数据库记录
- FAQ
- 代码文件
- 表格
- 聊天记录

## 文档解析

不同格式需要不同解析方式：

| 格式 | 解析重点 |
|---|---|
| Markdown | 保留标题层级和链接 |
| PDF | 处理页眉、页脚、换行、扫描件 OCR |
| Word | 保留标题、表格和列表 |
| HTML | 去除导航、广告、脚本 |
| Excel | 保留表头、单位和字段含义 |

## 清洗目标

清洗不是让文本变短，而是让文本更适合检索。

常见清洗动作：

- 去除重复页眉页脚。
- 修复异常换行。
- 删除目录、版权页、无意义符号。
- 保留标题、章节、来源信息。
- 将表格转换成可读文本。
- 统一编码和标点。

## 元数据

每个 Chunk 应尽量保存元数据：

- 文档标题
- 文件路径或 URL
- 作者
- 创建时间
- 更新时间
- 所属项目
- 章节标题
- 页码
- 权限标签

## 数据质量原则

垃圾数据进入知识库，就会产生垃圾答案。RAG 项目的第一优先级通常是数据治理，而不是调模型参数。
