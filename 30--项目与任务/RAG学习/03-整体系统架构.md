# 整体系统架构

一个完整 RAG 系统一般分为两条链路：离线建库链路和在线问答链路。

## 离线建库链路

离线链路负责把原始资料处理成可检索的知识库。

流程：

1. 收集文档。
2. 解析文档内容。
3. 清洗无效文本。
4. 按规则切分成 Chunk。
5. 为每个 Chunk 生成 Embedding。
6. 写入向量数据库。
7. 保存元数据，例如标题、来源、时间、权限。

## 在线问答链路

在线链路负责响应用户问题。

流程：

1. 接收用户问题。
2. 对问题做改写或扩展。
3. 检索相关 Chunk。
4. 可选：对结果进行重排序。
5. 组装上下文。
6. 调用大模型生成答案。
7. 返回答案和引用。

## 核心组件

| 组件 | 作用 |
|---|---|
| 文档解析器 | 从 PDF、Word、网页、Markdown 中提取文本 |
| Chunker | 将长文档切成适合检索的小块 |
| Embedding 模型 | 将文本转换成向量 |
| 向量数据库 | 存储和检索向量 |
| Retriever | 根据问题找相关内容 |
| Reranker | 对候选结果重新排序 |
| LLM | 生成最终回答 |
| Evaluator | 评估检索和回答质量 |

## 架构重点

RAG 的工程难点不在“调用一次模型”，而在让数据能被稳定、准确、低成本地检索和使用。
