# RAG 基本概念

## 什么是 RAG

RAG 是 Retrieval-Augmented Generation 的缩写，中文通常叫“检索增强生成”。它让大模型在回答问题前，先去外部知识库中检索相关内容，再基于检索结果生成答案。

## 为什么需要 RAG

大模型本身有几个天然限制：

- 知识可能过期。
- 对私有数据不了解。
- 容易在缺少依据时编造答案。
- 无法直接记住企业文档、个人笔记、产品手册等动态知识。

RAG 的作用就是把“模型能力”和“可控知识源”连接起来。

## 标准流程

1. 用户提出问题。
2. 系统将问题转换成向量，或提取关键词。
3. 从知识库中检索相关片段。
4. 将片段作为上下文放入 Prompt。
5. 大模型基于上下文生成答案。
6. 返回答案、引用来源或置信度信息。

## RAG 和微调的区别

| 对比项 | RAG | 微调 |
|---|---|---|
| 主要用途 | 注入外部知识 | 改变模型行为或风格 |
| 数据更新 | 快，更新知识库即可 | 慢，通常要重新训练 |
| 可解释性 | 较强，可返回来源 | 较弱 |
| 适合场景 | 问答、知识库、文档助手 | 特定格式、语气、任务习惯 |

## 关键思想

RAG 不是让模型“知道一切”，而是让模型“会查资料，并根据资料回答”。
