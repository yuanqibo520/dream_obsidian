# 流水线

## 基本思想

把指令执行过程划分为多个阶段，不同指令在不同阶段重叠执行，以提高吞吐率。

经典五级流水线：

1. IF：取指。
2. ID：译码/读寄存器。
3. EX：执行/地址计算。
4. MEM：访存。
5. WB：写回。

## 性能

理想情况下，k 段流水线执行 n 条指令所需时间约为：

```text
(k + n - 1) × 时钟周期
```

## 流水线冲突

### 结构冲突

多个阶段争用同一硬件资源。

### 数据冲突

后续指令需要前面指令尚未产生的结果。

### 控制冲突

分支指令导致下一条指令地址不确定。

## 解决方法

- 增加硬件资源。
- 数据转发。
- 流水线暂停。
- 分支预测。
- 延迟分支。

## 常考点

流水线提高吞吐率，不一定缩短单条指令执行时间。
