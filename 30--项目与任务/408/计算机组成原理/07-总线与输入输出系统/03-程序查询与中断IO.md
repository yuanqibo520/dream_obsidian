# 程序查询与中断 I/O

## 程序查询方式

CPU 反复查询外设状态，外设准备好后再传输数据。

优点：

- 实现简单。

缺点：

- CPU 等待时间长，效率低。

## 中断方式

外设准备好后主动向 CPU 发出中断请求，CPU 响应后执行中断服务程序完成数据传输。

优点：

- CPU 不必一直等待外设。
- 提高 CPU 利用率。

缺点：

- 每次传输仍需 CPU 参与。
- 中断处理有额外开销。

## 对比

| 项目 | 程序查询 | 中断 I/O |
|---|---|---|
| CPU 是否主动等待 | 是 | 否 |
| 实现复杂度 | 低 | 中 |
| CPU 利用率 | 低 | 较高 |
| 适用场景 | 简单低速设备 | 随机低速事件 |

## 常考点

中断方式适合外设速度慢、事件不连续的场景。
