### 消息卡片绑定（仅支持 Cici 和飞书）
>**基本设置**
- 成功显示小绿点
- 智能体以卡片的形式回复
- ![[Pasted image 20251221172819.png]]
- 发布后显现
>**设计卡片**
- 通过拖拽图片设计，可编辑基本文本大小
- 可转化电脑端和移动端
- **点击事件**：当用户点击卡片上的某个区域（如按钮、标题、图片等）时，触发的**预设操作**
	- **打开链接（Open URL）**
	- **发送消息（Send Message）
	- **调用插件（Call Plugin）**
- **循环渲染**：**当插件返回的是一个列表（数组）数据时，卡片能自动为每一项生成一个子卡片或重复区块**，而无需手动写多条
- **显隐设置**（显示/隐藏条件）是指：**根据插件返回的数据内容，决定某个元素是否显示**。
	- 条件逻辑较简单，不支持复杂表达式（如 A && B || C）
	- 显隐判断通常基于 字段是否存在且非空（null、""、undefined 视为隐藏）
- **变量**
	- 是插件或者工作流等将数据传递至卡片的桥梁开发者可以自己定义变量类型
	- 或将插件或工作流返回的数据结构定义在卡片中然后将需要动态渲染的字段在组件的属性中关联

| 部分      | 含义                         | 举例                        |
| ------- | -------------------------- | ------------------------- |
| **左值**  | 要判断的**数据字段**（来自插件返回的数据）    | `discount`、`price`、`type` |
| **操作符** | 比较方式                       | `==`!=`、`>`、`非空`、`为空` 等   |
| **右值**  | 用来比较的**目标值**（可以是固定值或另一个字段） | `"movie"`、`0`、`""`（空字符串）  |

### 💡 提示
>如果操作符是 “非空”/“为空”，通常不需要填右值
>字符串值一般**不用加引号**，Coze 会自动识别

### 参数设置
- **默认值**：防止因用户未提供关键参数导致调用失败
- **可见性（Enable 开关）**：
	- **开启**：LLM 可见该参数，会基于上下文决定是否覆盖默认值。
	- **关闭**：LLM 不可见，强制使用默认值，避免冗余判断。

### **智能体无法调用插件？**
分析角度
- 所用 LLM 是否支持 **Function Call** 功能；
- Prompt 是否清晰说明**何时调用插件**；
- 对话中是否能正确提取插件所需参数

# 工作流

#### 前提条件
- 该工作流必须已在你的**团队工作区资源库**中创建并发布。
#### 节点（Nodes）——工作流的核心

| 节点类型                        | 用途                                             |
| --------------------------- | ---------------------------------------------- |
| **LLM 节点**                  | 调用大模型生成文本、推理、总结等                               |
| **代码节点**（Code Node）         | 执行自定义 Python/JavaScript 逻辑                     |
| **插件节点**（Plugin Node）       | 调用平台插件（如搜索、翻译、天气）                              |
| **数据库节点**                   | 包括：添加/查询/更新/删除数据（Add/Query/Update/Delete Data） |
| **问答节点**（Question Node）     | 向用户提问并获取输入（常用于 Chatflow）                       |
| **变量赋值节点**（Variable Assign） | 将值存入变量                                         |
| **条件节点**（Condition）         | 实现 if/else 分支逻辑                                |
| **循环节点**（Loop / Batch）      | 批量或循环处理数据                                      |
| **子工作流节点**（Workflow Node）   | 调用其他工作流，实现模块化                                  |
- 类型
- 所有节点均支持多种数据类型，无需额外转换
- 适用场景：自动化、批处理、无交互或单次交互任务
- 工作流和对话流可相互转化：
		区别在于能否开始（话题名称）+（输入）
		LM 节点获得读取聊天历史的能力
	






疑问
- OAuth 插件是什么鬼
	- 一类统称需要用户授权才能访问第三方服务账户数据保障安全的插件
	- 邮箱与通讯、日历与日程、社交与内容、文档与知识管理
- Function Call是什么
	- 函数调用，通过function call来调用（插件、API、工作流）
	- 自动，自定义得自己配置
- 如何判断是否支持
	- 添加插件即有
- 消息卡片的形式
	- 一种**结构化、美观的回复形式**，用于让智能体在聊天界面中以**富媒体卡片**（而非纯文本）展示信息，提升用户体验和信息传达效率。
		- **结构化展示数据**
		- **支持交互操作**：部分卡片可带按钮（如“查看详情”“确认预订”）
		- **提升专业感与可读性**：比大段文字更清晰直观
		- **仅支持特定客户端**
- Coze 还支持将 AI 应用以 API 或 Web SDK 形式部署
