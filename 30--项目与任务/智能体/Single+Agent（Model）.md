## 模型设置
| 模式                    | 特点                              | 适用场景                       |
| --------------------- | ------------------------------- | -------------------------- |
| **Precise（精准）**       | 严格遵循指令，用词重复，话题集中                | 售后客服、事实问答                  |
| **Balance（平衡）**       | 随机性与准确性均衡                       | 通用聊天、日常助手                  |
| **Creative（创意）**      | 回答多样、新颖，可能偏离主题                  | 写作、头脑风暴                    |
| 参数==不建议同时调多个==        | 作用                              | 建议                         |
| **Temperature**       | 控制随机性<br>↑ 值 → 更创意；↓ 值 → 更确定    | 事实问答：0.2~0.5<br>创作：0.7~1.0 |
| **Top P**             | 控制词汇选择范围<br>只从累计概率达 Top P 的词中抽样 | 通常 0.8~0.95                |
| **Frequency Penalty** | 惩罚重复用词 → 增加词汇多样性                | +0.5 ~ +1.0                |
| **Presence Penalty**  | 惩罚重复话题 → 鼓励话题切换                 | +0.5 ~ +1.0                |

### 4. **默认指令（Default Instruction）**

开启后自动追加系统指令，提升稳定性：

- ✅ **当前时间**：每次用户提问自动附带准确时间
- ✅ **防 Prompt 泄露**：强化系统提示，降低模型泄露内部指令的风险
## 5. 深度思考（Deep Thinking） 
⭐（仅 Gemini 2.5 Pro 支持）
>AI 先进行 链式推理（Chain-of-Thought），再输出答案 → 更准确
>(🔔 适合：需要高精度推理但无需外部工具的场景（如数学题、逻辑分析）)
- 开启后：
	- ❌ 禁用所有插件、变量、数据库、工作流、Function Call
	- ⚠️ 消耗更多 tokens
	- but 可设置 思考预算（Thinking Budget Tokens），最低 1024 tokens


## 疑问
- Top P
- Presence Penalty和Frequency  Penalty（国际版独有）
	- Presence Penalty->话题->推动 AI 切换话题或角度，增加内容广度
	（💡 适合：写文案、创作、避免 AI“卡壳重复”）
	- Frequency  Penalty->词汇->增加词汇多样性，避免啰嗦重复
	（💡 适合：头脑风暴、多角度回答、避免钻牛角尖）
- token
	- ai能够理解语句的最小片段
	- 输出比输入的价格更高
	- solution：
		- 在模型设置中 调低“最大响应长度”
		- 简化prompt
		- 用 知识库（RAG） 替代全文输入
		- 复杂任务拆成多步（比如先总结，再分析）
		- 切换限制更高、消耗更小的ai模型
- 上下文管理
	- 把前面的对话当作“背景信息”，用于理解当前问题
	- 有时间限制，关闭聊天或3天内自动销毁
- JSON
	- 是一种**轻量级的数据交换格式**，用来**存储和传输结构化数据**。它易于人阅读和编写，也易于程序解析和生成。
	- 几乎所有编程语言都支持解析 JSON（如 Python、JavaScript、Java 等）

| 格式       | 本质     | 主要用途                |
| -------- | ------ | ------------------- |
| JSON     | 数据格式   | 存储/传输结构化数据（给程序用）    |
| Markdown | 文本标记语言 | 写带格式的文档（给人看 + 简单排版） |
