# 调试与测试

## 调试方法

- 打断点观察变量。
- 单步执行确认流程。
- 查看调用栈定位路径。
- 使用日志记录关键状态。

## 常见调试工具

- GDB。
- LLDB。
- Visual Studio Debugger。

## 单元测试

测试用于确认函数或模块行为符合预期。

```cpp
int add(int a, int b) {
    return a + b;
}
```

测试目标：

- 正常输入。
- 边界输入。
- 错误输入。

## Sanitizer

编译时可开启 Sanitizer 检查内存错误。

```bash
g++ main.cpp -fsanitize=address -g
```

## 建议

出现 bug 时先构造最小复现，再定位根因。
