# 封装

## 核心思想

封装是把数据隐藏起来，只暴露稳定、清晰的接口。

```cpp
class BankAccount {
public:
    void deposit(int amount) {
        if (amount > 0) {
            balance_ += amount;
        }
    }

    int balance() const {
        return balance_;
    }

private:
    int balance_ = 0;
};
```

## 不变量

不变量是对象始终应该满足的条件。例如账户余额不能因为非法操作变成负数。

## 好处

- 减少外部代码依赖内部细节。
- 方便修改实现。
- 降低错误传播范围。

## 建议

成员变量通常设为 `private`，通过函数维护对象状态。
