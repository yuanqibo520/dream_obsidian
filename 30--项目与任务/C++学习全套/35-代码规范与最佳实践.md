# 代码规范与最佳实践

## 可读性

- 名字表达意图。
- 函数保持短小。
- 一个函数做一件清晰的事。
- 避免深层嵌套。

## 类型安全

- 使用 `enum class` 替代裸 `enum`。
- 使用 `constexpr` 替代简单常量宏。
- 使用 `static_cast` 替代 C 风格转换。

## 资源安全

- 优先使用 RAII。
- 默认使用 `std::unique_ptr` 表达所有权。
- 避免手写 `new` 和 `delete`。

## 性能意识

- 先写正确清晰的代码，再测量瓶颈。
- 大对象传参用 `const T&`。
- 合理使用移动语义。

## 工程习惯

- 打开编译警告。
- 使用格式化工具。
- 为关键逻辑写测试。
- 代码提交保持小而清晰。
