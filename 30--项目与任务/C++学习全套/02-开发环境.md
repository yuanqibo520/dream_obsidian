# 开发环境

## 编译器

常见 C++ 编译器：

- GCC：Linux 常用，命令是 `g++`。
- Clang：错误提示友好，跨平台能力强。
- MSVC：Windows Visual Studio 默认编译器。

## 命令行编译

```bash
g++ main.cpp -std=c++20 -Wall -Wextra -O2 -o app
./app
```

参数含义：

- `-std=c++20`：使用 C++20 标准。
- `-Wall -Wextra`：打开常见警告。
- `-O2`：开启优化。
- `-o app`：指定输出文件名。

## IDE 和编辑器

- Visual Studio：适合 Windows **大型 C++ 项目**。
- CLion：跨平台，**CMake 支持好。**
- VS Code：轻量，需要配置**编译器和插件***。

## 建议

初学时先会命令行编译，再使用 IDE。这样能理解编译、链接、运行的真实过程。


# 拓展 是否需要掌握CMAKE
对于CMAKE需要掌握，但**不用一开始就深入掌握**。

如果你现在是在学 C++，建议这样看 CMake：

1. **初学阶段**
   只要知道它是“管理 C++ 项目编译的工具”即可。你会写单个 `.cpp` 文件、会用编译器运行程序，比学 CMake 更重要。

2. **开始写多个文件时**
   比如有：

```text
main.cpp
student.cpp
student.h
```

这时就值得学一点 CMake，因为它可以帮你统一管理这些文件怎么编译、怎么链接。

3. **做正式项目 / 看开源项目时**
   CMake 基本就很重要了。很多 C++ 开源项目都会有 `CMakeLists.txt`，你需要知道怎么配置、构建、运行。

你可以先掌握这些就够用：

```cmake
cmake_minimum_required(VERSION 3.10)
project(MyProject)

set(CMAKE_CXX_STANDARD 17)

add_executable(MyProject main.cpp)
```

以及常用命令：

```bash
cmake -S . -B build
cmake --build build
```

结论：**学 C++ 初期不必被 CMake 卡住；但如果你想正经写 C++ 项目，CMake 迟早要掌握基础用法。**
