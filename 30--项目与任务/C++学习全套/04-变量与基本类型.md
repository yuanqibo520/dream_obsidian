# 变量与基本类型

## 变量

变量是有名字的存储位置。

```cpp
int age = 18;
double price = 19.9;
char grade = 'A';
bool ok = true;
```

## 常见基本类型

- `int`：整数。
- `long long`：更大范围的整数。
- `float`：单精度浮点数。
- `double`：双精度浮点数。
- `char`：字符。
- `bool`：布尔值。

## const

`const` 表示不可修改。

```cpp
const double pi = 3.14159;
```

## auto

`auto` 让编译器推导类型。

```cpp
auto count = 10;      // int
auto name = "cpp";    // const char*
```

## 类型转换

```cpp
double x = 3.8;
int y = static_cast<int>(x);
//static_cast是静态类型转化

```

优先使用 `static_cast`，少用 C 风格强制转换。


# 扩展 

### 问题1：有没有动态转化
有的，但不是 `auto`。
	
在 C++ 里，和 `static_cast` 对应的“动态类型转换”通常是：

```cpp
dynamic_cast<T>(expr)
```

比如：

```cpp
// 定义一个基类 Base
class Base {
public:
    // 虚析构函数
    // 作用：
    // 1. 让 Base 成为“多态类型”，这样才能使用 dynamic_cast
    // 2. 以后通过 Base* 删除 Derived 对象时，能正确调用析构函数
    virtual ~Base() = default;
};

// 定义一个派生类 Derived，它继承自 Base
class Derived : public Base {
public:
    // Derived 自己的成员函数
    void hello() {
        // 这里暂时什么也不做
    }
};

// 创建一个 Derived 对象
// 但是用 Base* 类型的指针 p 来保存它
// 这叫“基类指针指向派生类对象”
Base* p = new Derived;

// 尝试把 Base* 类型的指针 p 转换成 Derived* 类型
// dynamic_cast 会在运行时检查：
// p 实际指向的对象到底是不是 Derived
Derived* d = dynamic_cast<Derived*>(p);

// 如果转换成功，d 就不是 nullptr
// 如果转换失败，d 就是 nullptr
if (d) {
    // 转换成功后，就可以调用 Derived 类自己的函数
    d->hello();
}
```

这里 `dynamic_cast` 会在运行时检查：`p` 实际指向的对象是不是 `Derived`。如果是，转换成功；如果不是，指针转换结果是 `nullptr`。

如果是引用：

```cpp
// p 是一个 Base* 指针
// *p 表示取出 p 指向的那个对象本身
// b 是 Base&，也就是 Base 类型的引用
//
// 注意：
// 这里虽然 b 的类型是 Base&，
// 但如果 p 实际指向的是 Derived 对象，
// 那么 b 引用的真实对象仍然是 Derived
Base& b = *p;

try {
    // 尝试把 Base& 类型的引用 b
    // 转换成 Derived& 类型的引用
    //
    // dynamic_cast 会在运行时检查：
    // b 引用的真实对象到底是不是 Derived
    //
    // 如果真实对象是 Derived，转换成功
    // d 就可以当作 Derived 引用来使用
    Derived& d = dynamic_cast<Derived&>(b);

    // 如果转换成功，可以调用 Derived 自己的函数
    d.hello();

} catch (const std::bad_cast& e) {
    // 如果引用转换失败，不会返回 nullptr
    // 因为引用不能是空的
    //
    // 所以 dynamic_cast<Derived&>(b)
    // 转换失败时会抛出 std::bad_cast 异常
}
```

引用转换失败会抛出 `std::bad_cast`。

`auto` 不是类型转换，它是**自动类型推导**：

```cpp
auto x = 10;      // x 是 int
auto y = 3.14;    // y 是 double
```

它发生在编译期，不会做运行时类型检查。

简单对比：

```cpp
static_cast<Derived*>(p);   // 编译期认为可以转，运行时不检查
dynamic_cast<Derived*>(p);  // 运行时检查真实对象类型
auto x = expr;              // 根据表达式推导变量类型
```

注意：`dynamic_cast` 通常要求基类是多态类型，也就是至少有一个 `virtual` 函数，常见做法是给基类写虚析构函数。

### try和catch是什么玩意
`catch` 不是函数名，也不是像 `cout` 那样的对象/固定函数。

它是 C++ 里的**关键字**，专门和 `try` 一起用于**异常处理**。

```cpp
try {
    // 这里放可能出错、可能抛异常的代码
    Derived& d = dynamic_cast<Derived&>(b);
}
catch (const std::bad_cast& e) {
    // 如果 try 里面抛出了 std::bad_cast 类型的异常
    // 程序就会跳到这里处理
}
```

这句：

```cpp
catch (const std::bad_cast& e)
```

可以理解为：

> 如果前面的 `try` 代码块里抛出了一个 `std::bad_cast` 类型的错误，就把它用引用的方式命名为 `e`，然后进入 `{}` 里面处理。

拆开看：

```cpp
catch ( const std::bad_cast& e )
```

- `catch`：关键字，表示“捕获异常”
- `std::bad_cast`：异常类型
- `&`：引用，避免复制异常对象
- `const`：不修改这个异常对象
- `e`：变量名，可以自己改，比如 `err`

比如：

```cpp
catch (const std::bad_cast& err) {
    // err 也是可以的
}
```

所以它不是函数调用，虽然长得有点像函数参数列表。它是 C++ 语法的一部分。
