# C++ 编译流程详解

## 总流程

C++ 从源代码到可执行文件通常经过四步：

1. 预处理。
2. 编译。
3. 汇编。
4. 链接。

## 预处理

预处理器处理以 `#` 开头的指令。

```cpp
#include <iostream>
#define DEBUG
```

预处理阶段会展开头文件、处理宏、处理条件编译。

可以用类似命令观察预处理结果：

```bash
g++ -E main.cpp -o main.i
```

## 编译

编译阶段把预处理后的 C++ 代码转换成汇编代码，并进行语法检查、类型检查和优化。

```bash
g++ -S main.i -o main.s
```

常见编译期错误包括：

- 语法错误。
- 类型不匹配。
- 找不到声明。
- 模板实例化失败。

## 汇编

汇编阶段把汇编代码转换成目标文件。

```bash
g++ -c main.s -o main.o
```

目标文件中包含机器码，但它通常还不是完整可执行程序。

## 链接

链接阶段把多个目标文件和库组合成最终可执行文件。

```bash
g++ main.o utils.o -o app
```

常见链接错误包括：

- 声明了函数但没有提供定义。
- 同一个函数或全局变量被重复定义。
- 使用了第三方库但没有正确链接。

## 头文件与源文件的关系

头文件通常放声明，源文件通常放定义。

```cpp
// math.h
int add(int a, int b);
```

```cpp
// math.cpp
#include "math.h"

int add(int a, int b) {
    return a + b;
}
```

```cpp
// main.cpp
#include "math.h"

int main() {
    return add(1, 2);
}
```

编译时每个 `.cpp` 通常独立生成目标文件，链接时再合并。

## 学习重点

- 编译错误通常说明单个翻译单元内部的问题。
- 链接错误通常说明多个文件或库之间的符号关系有问题。
- 头文件不要随意放普通函数或全局变量定义，否则容易重复定义。
- 理解编译流程后，CMake、库链接和工程报错会容易很多。
