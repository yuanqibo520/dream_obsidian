## 1. 运算符和控制语句

### 1.1 运算符结合性

| 结合类型 | 包含运算符                                                                                                                                                 | 说明                  |
| ---- | ----------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------- |
| 右结合  | 赋值类：`=` `+=` `-=` `*=` `/=` 等<br><br>单目运算符：`++` `--` `!` `~` `+` `-` `*` `&` `(type)`<br><br>三目运算符：`exp1?exp2:exp3`<br>通过判断epx1得到2或3<br><br>函数调用：`()` | `++`和`*`优先级相同，遵循右结合 |
| 左结合  | 除单目、赋值、三目外的多数运算符（如算术、关系、逻辑运算符）                                                                                                                        | 从左到右依次计算            |

### 1.2 `sizeof` 用法
- `sizeof(type)`：计算指定类型的字节长度
- `sizeof(arr[0])`：计算数组单个元素的字节数
- `sizeof(arr)`：计算整个数组的字节数
- 数组元素个数计算公式：`sizeof(arr) / sizeof(arr[0])`
### 1.3 条件 / 分支语句

- `if`/`while`/`do-while`：判断条件非 0 为真、0/NULL 为假
- `switch`：多分支匹配，需注意`break`使用
```C
switch(void type)
{
	case 1/'':
	break;
	default :
	;
}
```
- `goto`：跳转语句（慎用），示例：
~~~C
again: 
printf("hello, world"); 
goto again; // 跳转到again标签处
~~~
- 逗号表达式：
    - 从左到右依次计算每个子表达式；
    - 整个表达式的值为最后一个子表达式的值。
    - **输出结果**：`res=9, a=2, b=4, c=3`
```C
#include <stdio.h> 
int main() { 
int a = 1, b = 2, c = 3; // 逗号表达式：从左到右执行，结果为最后一个表达式的值 
int res = (a++, b += 2, c * 3); 
printf("res=%d, a=%d, b=%d, c=%d\n", res, a, b, c); return 0; }
```
### 1.4 输入输出与系统函数

#### 1.4.1 输入函数（`<stdio.h>`）

- `scanf`：` %d` 读取数字前会跳过空白字符（空格、`\t`、`\n`等）
- `getchar`：读取单个字符（包括`\n`），常用清空缓冲区：
~~~C
int c; 
while ((c = getchar()) != '\n' && c != EOF); // 清空一行输入（EOF=-1）
~~~
`fgets`：安全读取字符串，原型：
~~~C
char *fgets(char *str, int size, FILE *stream);
fgets(arr, sizeof(arr), stdin);
arr[strcspn(arr, "\n")] = '\0';
~~~
- `str`：存储读取内容的字符数组；
- `size`：最多读取`size-1`个字符（预留`\0`位置）；
- `stream`：输入源（如`stdin`）；
- 消除换行符：`buf[strcspn(buf, "\n")] = '\0';`（`strcspn`返回第一个`\n`的下标）。

## 2. 字符串与内存操作

### 2.1 字符串函数（`<string.h>`）
| 函数                                                             | 功能               | 注意事项                                                                                                                           |
| -------------------------------------------------------------- | ---------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `strlen`                                                       | 计算字符串长度（遇`\0`停止） | 返回值为`size_t`（无符号整型），不支持链式比较                                                                                                    |
| `strcpy(arr,"dawddawdw")`                                      | 字符串拷贝            | 源字符串需以`\0`结尾，目标数组内存需足够大                                                                                                        |
| `strcat`                                                       | 字符串追加            | 源字符串需以`\0`结尾                                                                                                                   |
| `strcmp`                                                       | 字符串比较            | 左 <右返回负数，左 = 右返回 0，左> 右返回正数                                                                                                    |
| `strncmp`/`strncat`/`strncpy`<br>`strncmp(arr1,arr2,size_num)` | 限定长度的字符串操作       | - `strncmp`：`size_num>strlen(arr2)`时，用`\0`补充<br><br>- `strncat`：`size_num>strlen(arr2)`时不补充，自动追加`\0`<br><br>- `strncpy`：限定拷贝长度 |
| `strstr（索引地址，目标内容）`                                            | 查找子串             | 返回子串首地址（未找到返回 NULL）                                                                                                            |
### 2.2.1 内存函数（`<string.h>`）
| 函数                                                   | 功能     | 核心说明                                                                                                                 |
| ---------------------------------------------------- | ------ | -------------------------------------------------------------------------------------------------------------------- |
| `memcpy(void* dest, void* src, size_t num)`          | 内存拷贝   | `num`为字节数（非元素个数），不处理重叠内存                                                                                             |
| `memmove(void* dest, void* src, size_t num)`         | 重叠内存拷贝 | 目标指针 < 源指针：从前向后拷；反之从后向前拷                                                                                             |
| `memcmp(const void* p1, const void* p2, size_t num)` | 内存比较   | 逐字节比较，不遇`\0`停止，返回规则同`strcmp`                                                                                         |
| `memset(void* ptr, int value, size_t num)`           | 内存初始化  | - 按字节赋值，仅推荐设置 0/-1（非 0 / 非 - 1 整型会出错）<br><br>- 示例：`memset(arr, 1, sizeof(arr))`会将 int 元素设为`0x01010101`（十进制 16843009） |
### 2.2.2 参数说明
| 参数      | 含义                      |
| ------- | ----------------------- |
| `ptr`   | 内存起始地址（数组、结构体、指针等）      |
| `value` | 字节值（仅取低 8 位，常用 0/-1）    |
| `num`   | 要设置的字节总数（推荐用`sizeof`计算） |
| size_t  | unsigned int            |
| dest指针  | 可能在其中被修改，用第三指针保存，再放回    |


### 2.3 核心区别

| 函数类型  | 停止条件     | 适用场景                 |
| ----- | -------- | -------------------- |
| 字符串函数 | 遇`\0`停止  | 文本字符串操作              |
| 内存函数  | 按指定字节数停止 | 结构体、二进制数据、数组等任意内存块操作 |

## 4. 程序环境与预处理

### 4.1 编译与链接流程

1. 预处理 → 编译 → 汇编 → 链接；
2. 预处理：生成`.i`文件，处理`#include`/`#define`、删除注释；
3. 编译：生成`.s`文件，将 C 代码转为汇编代码（语法 / 词法分析、符号汇总）；
4. 汇编：生成`.obj`文件，将汇编指令转为二进制指令（生成符号表）；
5. 链接：合并段表、符号表重定位，生成`.exe`可执行文件。

### 4.2 GCC 编译命令

|命令|功能|
|---|---|
|`gcc file.c -E -o file.i`|仅执行预处理|
|`gcc file.c -S`|预处理 + 编译，生成`.s`汇编文件|
|`gcc file.c -c`|预处理 + 编译 + 汇编，生成`.obj`目标文件|

### 4.3 预处理指令

#### 4.3.1 宏定义（`#define`）

- 无类型限制，文本替换，不可递归，调试难度高；


    ```c
    #define SQUARE(x) (x)*(x) // 避免副作用，需加括号
    #define PRINT(a) printf("a=%d", a) // 基础宏
    #define STR(a) #a // 把参数转为字符串（如STR(123)→"123"）
    #define JOIN(a,b) a##b // 拼接标识符（如JOIN(ab,cd)→abcd）
    #undef SQUARE // 取消宏定义
    ```

#### 4.3.2 条件编译

- `#if`/`#elif`/`#else`/`#endif`：条件判断；
- `#ifdef`/`#ifndef`：判断宏是否定义；

    ```c
    #ifndef DEBUG // 若未定义DEBUG宏
        #define DEBUG 1
    #endif
    ```

#### 4.3.3 `static` 与 `extern`

- `static`：
    
    - 局部变量：生命周期延长至程序结束，作用域仍在函数内；
    - 全局变量 / 函数：仅当前文件可见，不可跨文件调用；
    
- `extern`：声明外部变量 / 函数（告知编译器 “定义在别处”），示例：
    ```C
    extern int a; // 声明（不分配内存）
    int a = 10;   // 定义（分配内存）
    ```

## 5. 基本数据类型与存储

### 5.1 基础类型与格式化 IO

#### 5.1.1 数据类型大小（32/64 位通用）

|格式化符|说明|
|---|---|
|`%u`|无符号整数|
|`%ld`|长整型|
|`%x`/`%X`|十六进制（小写 / 大写）|
|`%02d`|占 2 位，不足补 0|
|`%.2f`|浮点数保留 2 位小数|
|`%%`|输出`%`符号|
|`\ddd`|八进制字符（如`\101`='A'）|
|`\xhh`|十六进制字符（如`\x41`='A'）|
#### 5.1.2 转义字符

| 转义字符      | 含义                   |
| --------- | -------------------- |
| `\n`      | 换行                   |
| `\t`      | 水平制表符（Tab）           |
| `\r`      | 回车                   |
| `\b`      | 退格                   |
| `\\`      | 反斜杠                  |
| `\'`/`\"` | 单 / 双引号              |
| `\0`      | 空字符（字符串结束标志，ASCII=0） |
| `%%`      | 输出一个 % 符号            |
### 5.2 原码、反码、补码

- 原码：二进制表示（最高位为符号位，0 正 1 负）；
- 反码：正数与原码相同；负数符号位不变，其余位取反；
- 补码：正数与原码相同；负数 = 反码 + 1（内存中存储的是补码）；
- 位运算（按补码计算）：
    
    - `&`：按位与（有 0 则 0）；
    - `|`：按位或（有 1 则 1）；
    - `^`：按位异或（相异则 1）。
    

### 5.3 常量与存储

- `const` 与 `#define` 对比：

|特性|`const`|`#define`|
|---|---|---|
|类型|有类型|无类型|
|作用域|局部 / 全局|全局（文本替换）|
|内存|分配内存（可取地址）|不分配内存|
|特性|只读变量|宏常量|

- 大小端存储：
    
    - 小端：低字节存低地址（x86 架构默认）；
    - 大端：低字节存高地址；
    - 判断方法：取`int a=1`的首字节，解引用为 1 则小端，0 则大端。
    

### 5.4 实用技巧

#### 5.4.1 无临时变量交换值（异或法）
```C
a = a ^ b;
b = a ^ b; // b=原a
a = a ^ b; // a=原b
```

#### 5.4.2 `const` 指针

- `const int *p`：指针可改，不可通过`*p`修改值；
- `int *const p`：指针不可改，可通过`*p`修改值；
- `const int *const p`：指针和值都不可改。

	- float x64位，1位符号，11位指数，52位有效值，V=(-1)^E * M* (2)^E

## 6. 结构体、枚举与联合

### 6.1 结构体

- 声明结构体为一个档案带，得预设其中的所有内容，在主函数中定义结构体变量或者数组，也可以在结构体尾处定义，可以在函数内部声明节省内存
- 结构体嵌套为在父结构体内部定义一个结构体变量， 子结构体必须**先定义**或**前向声明**（若仅用指针可前向声明）
#### 6.1.1 基础用法

- 定义与初始化：
    ```C
    struct Point {
        int x;
        int y;
    } p1; // 声明时定义全局变量
    struct Point p2 = {.x=1, .y=2}; // 指定成员初始化
    ```
- `typedef` 简化类型：
    ```C
    typedef struct Point Point;
    Point p3; // 无需写struct Point
    typedef unsigned long ulong; // 简化基本类型
    ```
- 结构体赋值：C99 及以上支持同类型结构体直接赋值（`s1 = s2`）；
- 字符串赋值：需用`strcpy`，不可直接`=`；
- 结构体指针：`ptr->member`（等价于`(*ptr).member`）。

#### 6.1.2 内存对齐（核心规则）

1. 首个成员对齐到结构体起始地址；
2. 其他成员对齐到 “对齐数”（默认对齐数（如 VS=8）与成员字节数的较小值）的整数倍；
3. 结构体总大小为最大对齐数的整数倍；
4. 嵌套结构体：对齐到自身最大对齐数的整数倍；

- 对齐原因：硬件访问效率（未对齐需 2 次访问，对齐仅 1 次）；
- 修改默认对齐数：`#pragma pack(8)`（括号内为对齐数）。

### 6.2 位段（内存优化）

- 仅支持`int`/`unsigned int`/`char`类型，示例：

~~~C
  struct BitSeg {
  int a:4; // 占4个比特位
  int b:8; // 占8个比特位
   };
~~~

- 跨平台问题：剩余比特位不足时，是否开辟新内存未统一规定，数据可能截断。
- 内容存储：剩余不够比特位则开辟新内存，无法确定是否舍去不够的存储空间
- 截断：存储数据内存大于开辟空间（数字），发生截断内存存储，编译器每次读取按四个比特位读取

### 6.3 枚举

- 定义：`enum Color {RED=1, GREEN, BLUE};`（默认从 0 开始，可手动赋值）；
- 优势：类型安全、可读性高、调试友好（优于`#define`）。

### 6.4 联合（共用体）

- 所有成员共享同一块内存，大小为最大成员的整数倍（需对齐），示例：
~~~C
    union Un {
        int i; // 4字节
        char c; // 1字节
    }; // 总大小4字节
~~~

## 6. 指针

### 6.1 基础概念

- 核心作用：修改函数外部变量的值；
- 野指针防范：初始化`int *p = NULL`，避免越界访问；
- `%p`：打印指针地址。

### 6.2 数组与指针

- `arr`/`&arr[0]`：数组首元素地址；
- `&arr`：整个数组的地址（类型为数组指针）；
- 二维数组：`a[i][j] = *(*(a+i)+j)`；
- 数组传参：
    
    - 一维数组：
	    - 一维数组传参int arr[10]，test（arr）
		- void test(int arr/int * arr/int arr[10]/int * arr[10]/int ** arr)
    - 二维数组：需指定列数，或用数组指针：`void test(int (*arr)[5])`。
    

### 6.3 指针数组 vs 数组指针

| 类型   | 写法            | 含义                      |
| ---- | ------------- | ----------------------- |
| 指针数组 | `int *p[5]`   | 存储指针的数组（5 个`int*`类型元素）  |
| 数组指针 | `int (*p)[5]` | 指向数组的指针（指向 5 个`int`的数组） |

### 6.4 函数指针

- 定义：`int (*pf)(int, int) = add;`（`add`为函数名）；
	-  Elemtype (* pf)(Elemtype1,Elemtype1)=add
- 特性：`&函数名`与`函数名`等价，调用时可省略`*`；
- 应用：回调函数（如`qsort`的比较函数）；
- `void*`：通用指针，可接收任意类型指针，使用前需强转。

## 7. 链表与数据结构基础

### 7.1 基础数据结构概念

- 栈：先进后出（`push`入栈、`pop`出栈），应用：进制转换、括号匹配；
- 队列：先进先出；
- 树：
    
    - 核心属性：根节点、高度（从节点到叶子的最大层数）、深度（从根到节点的层数）、度（子节点数）；
    - 二叉树：有序树，节点数 = 边数 + 1 = 所有节点度数之和 + 1。
    

### 7.2 链表操作

- 核心：动态内存分配（`malloc`/`free`）；
- 插入 / 删除：需辅助指针，避免断链；
- 注意：`free(p)`仅释放内存，需手动将`p=NULL`避免野指针；
- 头节点：常存储链表长度（数据域）。

### 7.3 动态数组（C++ 拓展）

~~~C
#include <vector> 
vector<int> v; v.push_back(2); // 添加元素 
auto it = find(v.begin(), v.end(), 2); // 查找元素 
v.erase(it); // 删除元素
~~~

## 8. 文件操作

### 8.1 核心函数（`<stdio.h>`）

#### 8.1.2 读写函数

- `fwrite`/`fread`（二进制读写）：
~~~c
size_t fwrite(const void* ptr, size_t size, size_t count, FILE* fp); 
size_t fread(void* ptr, size_t size, size_t count, FILE* fp);
~~~
**函数返回值都是次数，读取写入量为大小* 次数，即规定大小为1，次数为总量**
- 返回值：成功读写的 “块数”（总字节数 = size×count）；
    - `ptr`：数据缓冲区地址；
    - `fp`：文件句柄（`FILE*`），需先判断`fp!=NULL`。
    
- `fgets`/`fputs`（文本读写）：
    
    - `fgets`：最多读`n-1`个字符，遇`\n`/EOF 停止；

|参数|含义|
|---|---|
|`str`|字符数组 / 指针，用于存储读取到的字符串（会自动在末尾加 `\0` 终止符）|
|`n`|要读取的**最大字符数**（实际最多读 `n-1` 个字符，留 1 个位置给 `\0`）|
|`stream`|输入流：<br><br>- `stdin`：从键盘（标准输入）读取<br><br>- 文件指针（如 `fp`）：从文件读取

  ~~~ C
 #include <stdio.h> // 必须包含此头文件 
  char *fgets(char *str, int n, FILE *stream);
  ~~~

    - `fputs`：写入字符串，失败返回 EOF，成功返回非负数。

#### 8.1.3 文件定位与缓冲区

- `fseek(fp, offset, origin)`：移动文件指针
    
    - `origin`：`SEEK_SET(0)`（文件头）、`SEEK_CUR(1)`（当前位置）、`SEEK_END(2)`（文件尾）；
    
- `ftell(fp)`：返回指针偏移量；
- `fflush(fp)`：强制刷新缓冲区（写入磁盘）；
- `fclose(fp)`：关闭文件（必做，否则数据可能丢失）。
	- 成功返回0，失败返回非0
- strtok_s
	- 注意事项：
	- **分隔符是 “字符集合”，不是 “字符串”**
		- 比如 `strtok_s(str, " ,;", &ctx)` 表示 “遇到空格、逗号、分号都算分隔符”，而不是 “必须匹配 `,;` 这个字符串”。
	- **空字段的处理**
		- 如果遇到连续分隔符（比如 `张三,,zhangsan@xxx.com`），`strtok_s` 会跳过空字段，直接返回下一个非空字段（示例中会跳过空的电话字段，直接返回邮箱）
	- **context 必须初始化且全程复用**
```C
char* strtok_s( char* strToken, // 要分割的字符串（首次传待分割串，后续传 NULL） 
const char* strDelimit, // 分隔符（多个字符都算分隔符，比如 "," " \t\n"） 
char** context // 保存分割上下文的指针（关键：替代静态变量，保证安全） );
```

| 参数           | 作用                                                                          |
| ------------ | --------------------------------------------------------------------------- |
| `strToken`   | 首次调用：传入要分割的原始字符串（会被**原地修改**，替换分隔符为 `\0`）<br><br>后续调用：传入 `NULL`，表示继续分割同一个字符串 |
| `strDelimit` | 分隔符集合（比如 `","` 表示按逗号分割，`" ,\t\n"` 表示按逗号 / 空格 / 制表符 / 换行分割）                  |
| `context`    | 指向 `char*` 的指针，用来保存分割的 “上下文”（记录下次从哪里继续分割），**必须初始化**（比如 `char* ctx = NULL;`） |

## 9. 调试技巧

### 9.1 Debug vs Release

- Debug：包含调试信息，未优化，可断点调试；
- Release：无调试信息，编译优化，体积更小、运行更快。

### 9.2 调试快捷键

|快捷键|功能|
|---|---|
|F5|启动调试|
|Ctrl+F5|运行（不调试）|
|F9|切换断点（设置 / 清除）|
|F10|逐过程（跳过函数）|
|F11|逐语句（进入函数）|
|Ctrl+F10|运行到光标处|
|Shift+F5|停止调试|
### 9.3 调试工具

- 监视窗口：查看变量 / 数组（如`arr,10`显示数组前 10 个元素）；
- 调用堆栈：追溯函数调用链；
- `assert`：调试阶段验证逻辑，Release 版本可禁用。
