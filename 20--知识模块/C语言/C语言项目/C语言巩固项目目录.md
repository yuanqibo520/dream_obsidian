### 做项目的实用建议

1. 先做**菜单驱动式**结构（while+switch），养成好习惯
2. 能用文件保存的**尽量做文件读写**（这是C语言最实用技能之一）
3. 从第三四个项目开始，**强制自己拆分成多个.c/.h文件**
4. 第7个项目之后，**强烈建议使用动态内存**（malloc/free），不要再用大数组
5. 每个项目做完后尝试做**2~3个小升级**（加功能比重新做新项目更能提升）
6. 遇到卡点不要死磕超过2小时 → 看思路 → 自己重写（不要直接抄完整代码）
7. 建议使用**gcc + valgrind** 检查内存泄漏（尤其是动态内存项目）

### 1. 功能全面的计算器（支持括号、优先级）

**详细项目要求：**
- 支持基本的四则运算：加（+）、减（-）、乘（*）、除（/）。
- 支持括号（()）处理，以实现运算优先级。
- 支持负数和小数（浮点数）输入。
- 输入方式：从命令行读取一个完整的表达式字符串，例如 "3*(2+1)-4/2"。
- 需要处理运算符优先级（乘除高于加减，括号最高）。
- 实现方法：使用栈来实现中缀表达式转后缀表达式（逆波兰表示），然后计算后缀表达式。
- 错误处理：检测无效输入（如除零、括号不匹配），并输出错误消息。
- 额外要求：支持连续计算模式（循环输入表达式，直到用户输入"exit"退出）；使用函数模块化代码（如parse_expression、evaluate等）。
- 知识点考察：字符串处理、栈（数组实现）、递归或迭代、浮点运算、错误检查。

**实现效果描述：**
程序启动后，提示用户输入表达式。用户输入后，程序解析并计算结果，并显示。例如：
```
请输入表达式（输入exit退出）：3*(2+1)-4/2
结果：7.000000
```
如果输入无效，如"3/0"，输出"错误：除零"。支持复杂表达式如"(1.5 + 2.5) * (3 - 1)"，结果为"8.000000"。程序循环运行，直到退出。最终效果是一个命令行计算工具，能处理中等复杂度的数学表达式，类似于简易的科学计算器。

### 2. 猜数字游戏（带难度等级+历史记录）

**详细项目要求：**
- 游戏规则：程序随机生成一个1-100的数字，用户猜测，程序提示"太大"、"太小"或"正确"。
- 支持难度等级：简单（1-50，无限次）、中等（1-100，10次机会）、困难（1-500，8次机会）。
- 记录历史：使用文件保存每次游戏的难度、猜测次数、正确数字和结果（胜/负）。
- 输入方式：启动时选择难度，然后循环输入猜测数字。
- 错误处理：输入非数字或超出范围时提示重新输入。
- 额外要求：显示剩余机会；游戏结束后显示历史记录列表（读取文件）；支持重玩。
- 知识点考察：随机数（rand/srand）、循环、条件判断、文件读写（fopen/fprintf/fscanf）。

**实现效果描述：**
程序启动显示菜单："1.简单 2.中等 3.困难"。用户选择后，提示"猜一个数字（1-100）"。用户输入后反馈如"太小了，剩余9次"。猜对显示"恭喜！用了5次"。游戏结束显示历史，如：
```
历史记录：
游戏1: 中等, 用了5次, 正确数字:42, 胜
游戏2: 困难, 用了8次, 正确数字:233, 负
```
文件持久化，下次运行能加载历史。效果是一个互动小游戏，能记录多局数据，适合练习循环和文件操作。

### 3. 通讯录/电话簿（增删改查+文件持久化）

**详细项目要求：**
- 每个联系人包括：姓名（字符串）、电话（字符串）、邮箱（字符串）、地址（字符串）。
- 支持操作：添加、删除（按姓名）、修改（按姓名更新字段）、查询（按姓名或电话模糊搜索）、显示所有。
- 数据结构：使用结构体数组（最大100人），或简单链表。
- 文件持久化：启动时从文件加载，退出时保存到文件（如contacts.txt，格式为CSV）。
- 输入方式：菜单驱动（1.添加 2.删除 等），用户输入选项和数据。
- 错误处理：姓名重复、数组满时提示；搜索无结果时显示"未找到"。
- 额外要求：排序显示（按姓名A-Z）；支持导出到新文件。
- 知识点考察：结构体、数组、字符串操作、文件I/O、简单搜索。

**实现效果描述：**
程序显示菜单，用户选择"1"添加，输入姓名等信息。查询时输入姓名，显示详情如：
```
姓名: 张三
电话: 123456789
邮箱: zhang@example.com
地址: 北京
```
显示所有时列出列表，按姓名排序。文件保存为"张三,123456789,zhang@example.com,北京\n"。效果是一个小型数据库工具，能管理联系人列表，数据不丢失。

### 4. 学生成绩管理系统（多门课+排名）

**详细项目要求：**
- 每个学生包括：学号（int）、姓名（字符串）、成绩（3-5门课的float数组，如数学、英语、编程）。
- 支持操作：添加学生、删除（按学号）、修改成绩、查询单个、显示所有（带平均分和排名）。
- 数据结构：结构体数组（最大50人）。
- 文件持久化：从文件加载/保存（如students.csv）。
- 计算功能：每人平均分、全班排名（按平均分降序）、最高/最低分统计。
- 输入方式：菜单驱动，用户输入数据。
- 错误处理：学号重复、成绩范围（0-100）检查。
- 额外要求：支持按科目排序；计算班级平均分。
- 知识点考察：结构体数组、多维数组、排序（bubble/qsort）、文件。

**实现效果描述：**
菜单选择"显示所有"，输出表格如：
```
学号 姓名 数学 英语 编程 平均分 排名
001  李四 90   85   95   90.00  1
002  王五 80   90   85   85.00  2
```
查询单个显示详情。添加后自动保存文件。效果是一个学校管理系统原型，能处理成绩数据和排名，适合练习数组和排序。

### 5. 简易文本文件统计分析器

**详细项目要求：**
- 输入：指定文本文件路径（命令行参数或输入）。
- 功能：统计字符数、单词数、行数；最长/最短行；单词频率（前10高频词）。
- 处理：读取文件，逐行/逐字符分析；忽略标点，单词以空格分隔。
- 输出：显示统计结果，并可选保存到新文件。
- 错误处理：文件不存在、打开失败提示。
- 额外要求：支持大文件（逐行读）；忽略大小写统计词频（用数组或简单哈希）。
- 知识点考察：文件操作（fgets/fread）、字符串处理（strtok/strchr）、指针、数组计数。

**实现效果描述：**
运行程序输入文件"test.txt"，输出：
```
字符数: 1234
单词数: 567
行数: 89
最长行: 120字符
高频词: the(50), a(30), ...
```
效果是一个文件分析工具，像简化版的wc命令，能快速统计文本数据，练习文件和字符串操作。

### 6. 井字棋 / 五子棋（人vs人/人vs电脑）

**详细项目要求：**
- 棋盘：3x3井字棋或5x5五子棋（可选扩展）。
- 模式：人vs人，或人vs简单AI（随机或基本判断）。
- 操作：轮流输入坐标（行,列），检查胜负/平局。
- 显示：每次打印棋盘（用字符如X/O/-）。
- 胜负判断：横/竖/斜三连（井字）或五连（五子）。
- 额外要求：文件记录对局历史；AI用minimax（可选高级）。
- 错误处理：坐标无效、重下提示。
- 知识点考察：二维数组、循环判断、函数递归（AI）。

**实现效果描述：**
棋盘显示如：
```
X | O | -
---------
- | X | O
---------
- | - | X
```
轮到玩家："输入坐标(1-3,1-3)"。胜负后显示"X胜！"并记录。效果是一个命令行游戏，能多人玩或单人练习，练习数组和逻辑。

### 7. 单向链表版通讯录/待办事项（动态内存）

**详细项目要求：**
- 类似项目3，但用单向链表存储联系人/待办（描述、优先级、截止日期）。

### 10. 简易INI/配置文件解析器

**详细项目要求：**
- 输入：INI文件（如[section]\nkey=value）。
- 功能：解析成结构体/哈希（数组实现），支持读key值、修改、写回文件。
- 处理：跳过注释（;）、空行；支持多section。
- 输出：查询key返回value；修改后保存。
- 额外要求：默认值、错误key返回NULL。
- 知识点考察：文件、字符串分割（strtok）、指针数组、状态机。

**实现效果描述：**
解析文件后，get("section","key")返回"value"。修改后写回。效果是配置文件工具，像简易inih库，练习解析技能。
