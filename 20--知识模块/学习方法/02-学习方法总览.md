# 学习方法总览

这篇笔记用来统领当前“学习方法”目录。现有内容已经记录了很多有效的学习原则，例如 90/90/1、复盘、目标设定、鱼骨图、记忆方法、最小学习单元、刻意练习、作品思维和 TRIZ 理论。下一步需要把这些零散方法组织成一个可执行、可复盘、可迁移的学习系统。

## 学习闭环

完整的学习不只是“看懂内容”，而是一个循环：

1. 选择目标：明确要学什么，为什么学，学到什么程度。
2. 拆分任务：把大目标拆成每天能完成的最小学习单元。
3. 输入理解：阅读、听课、看案例，先建立基本框架。
4. 主动提取：不看资料复述、做题、画图、讲给别人听。
5. 输出作品：形成笔记、题解、项目、讲解稿、清单或模板。
6. 复盘改进：找到卡点、原因和下一次的调整动作。
7. 迁移应用：把知识用于新问题、新场景或真实任务。

## 方法地图

| 学习阶段 | 关键问题 | 可用方法 |
| --- | --- | --- |
| 学习前 | 学什么、为什么学、学到什么程度 | SMART 目标、鱼骨图、学习内容筛选 |
| 学习中 | 如何保持专注并真正理解 | 90/90/1、最小学习单元、图表和例题、费曼学习法 |
| 学习后 | 如何避免学完就忘 | 主动回忆、间隔重复、半小时关联整理 |
| 练习时 | 如何持续提高 | 刻意练习、拆解不足、针对性练习、微小改进 |
| 输出时 | 如何形成真正成果 | 作品思维、项目化学习、讲解型笔记 |
| 复盘时 | 如何发现问题原因 | 客观数据、视频回看、错题分析、鱼骨图 |
| 迁移时 | 如何用到新场景 | TRIZ、类比、案例库、跨学科连接 |

## 场景选择

### 自学一门技能

适合使用“90/90/1 + 最小学习单元 + 作品思维”。

执行方式：

- 选定 90 天内最重要的一项技能。
- 每天固定一段高质量时间，只推进一个核心方向。
- 每天产出一个可见成果，例如一段代码、一道题解、一页笔记、一个案例分析。
- 每周复盘一次：哪些内容真正掌握，哪些只是看过。

### 备考或刷题

适合使用“主动回忆 + 间隔重复 + 错题复盘”。

执行方式：

- 每个知识点都要能不看书复述。
- 错题不只记录答案，还要记录错误类型。
- 复习时间至少占总学习时间的一半。
- 把题目按场景、题型、陷阱和解法归类。

### 阅读理论书

适合使用“结构化笔记 + 费曼学习法 + 问题链”。

执行方式：

- 先看目录，判断全书在解决什么问题。
- 每章提炼 3 个问题：作者想解决什么、用了什么理由、能用在哪里。
- 用自己的话重写核心概念。
- 读完后输出一篇“我如何使用这本书”的行动清单。

### 学习编程或项目技能

适合使用“项目化学习 + 高频应用场景 + 复盘”。

执行方式：

- 不只看教程，要尽早做一个真实小项目。
- 每学一个概念，都对应一个可运行例子。
- 遇到 bug 时记录：现象、原因、定位过程、最终修复。
- 每周整理一份“下次遇到类似问题怎么处理”的经验卡。
