# 领导力

## 一句话定义

领导力不是职位带来的权力，而是让一群人愿意围绕共同目标行动，并把复杂事情持续推进出结果的能力。

真正的领导力 = 方向感 + 信任感 + 组织能力 + 承担责任 + 成人成事。

## 领导力解决什么问题

- 团队不知道为什么做。
- 下属不知道做到什么程度算好。
- 每个人都很忙，但事情没有往前走。
- 出问题时互相甩锅。
- 有能力的人不愿投入，普通人不知道怎么成长。

## 品格基础

### 1. 自我管理

要做：

- 管好自己的时间和精力。
- 管好自己的情绪反应。
- 对自己的任务和承诺有交代。

达到效果：

- 别人会觉得你稳定、可靠、值得跟随。
- 你说的话更有分量。

### 2. 勇于担当

担当不是把所有事都揽到自己身上，而是在关键时刻不逃避责任。

要做：

- 出问题时先稳住局面。
- 先复盘事实，再追责任。
- 对外保护团队，对内改进机制。

达到效果：

- 团队敢暴露问题，不会为了自保隐瞒风险。
- 你能建立真正的信任。

### 3. 以身作则

下属不会只听你说什么，会看你怎么做。

要做：

- 要求别人守时，自己先守时。
- 要求别人反馈，自己先反馈。
- 要求别人复盘，自己先复盘。

达到效果：

- 规则不靠压迫，而靠示范形成。
- 团队更容易建立共同标准。

### 4. 建立信任

信任来自长期稳定的行为。

要做：

- 承诺前谨慎，承诺后兑现。
- 公开表扬，私下纠偏。
- 不随意抢功，不轻易甩锅。

达到效果：

- 别人愿意跟你说真话。
- 关键时刻有人愿意支持你。

## 核心能力

### 1. 讲清方向

要做：

```text
我们为什么做这件事？
最终要解决什么问题？
优先级最高的是什么？
什么事情可以暂时不做？
```

达到效果：

- 团队不只是执行动作，而是理解目标。
- 遇到变化时，大家能自己判断取舍。

### 2. 拆清任务

要做：

```text
目标：
交付物：
验收标准：
负责人：
截止时间：
资源支持：
风险节点：
同步频率：
```

达到效果：

- 减少靠猜和返工。
- 每个人知道自己负责什么。

### 3. 会提问

推荐使用 GROW 模型：

```text
Goal：你想达成什么目标？
Reality：现在真实情况是什么？
Options：你有哪些选择？
Will：下一步你准备怎么做？
```

也可以用：

```text
目标是什么？
你现在怎么想？
你准备采取什么行为？
你预期结果是什么？
```

达到效果：

- 不直接替下属思考。
- 让对方在问题中成长。

### 4. 授权

授权不是放任，而是给目标、边界和资源。

要做：

- 讲清目标。
- 讲清不能突破的底线。
- 讲清需要同步的节点。
- 给对方方法自主权。

达到效果：

- 下属有空间成长。
- 你不会被所有细节拖住。

### 5. 纠偏

纠偏要对事不对人。

要做：

```text
指出具体行为。
说明造成的影响。
询问当时判断。
一起制定改法。
约定下次标准。
```

达到效果：

- 对方知道哪里错，而不是只感觉被否定。
- 团队保留尊严，也改进行为。

### 6. 激励

激励不是打鸡血，而是让人看见意义、进步和自己的价值。

要做：

- 夸具体贡献，不只夸态度。
- 让成员知道自己的工作如何影响整体结果。
- 给阶段性反馈和成长机会。

达到效果：

- 成员更愿意投入。
- 团队不只靠压力运转。

## 领导力训练表

| 训练动作 | 达到效果 |
| --- | --- |
| 每次安排任务前写清目标、标准、时间 | 下属少猜，执行更稳 |
| 每周追踪一次风险，不只追进度 | 提前发现问题，减少救火 |
| 公开表扬一次具体贡献 | 强化团队想要的行为 |
| 私下纠偏一次具体问题 | 保住尊严，同时修正行为 |
| 每周问下属一次“你需要什么支持” | 提升信任和协作质量 |
| 每次项目后做复盘 | 把经验沉淀成机制 |

## 领导力谚语与名言库

- 其身正，不令而行；其身不正，虽令不从。
- 上下同欲者胜。
- 将者，智、信、仁、勇、严也。
- 凡事预则立，不预则废。
- 管理是把复杂事情变清楚，把重要事情往前推。

## 领导沟通案例腹稿

### 分配任务

> 这件事的目标是 XX，验收标准是 XX，截止时间是 XX。你负责主推进，我会给你 XX 资源。中间如果遇到风险，不要等到最后一天再说，提前同步，我们一起处理。

### 追进度

> 我问进度不是为了催你，而是为了判断整体风险。你现在完成到哪一步？最卡的点是什么？如果只能优先保一个结果，你觉得应该先保哪一块？

### 批评下属

> 这次的问题我需要明确指出来：XX 行为导致了 XX 影响。我的重点不是责备你，而是希望下次不再重复。你先说说当时怎么判断的，我们再一起定一个改法。

### 激励团队

> 这件事确实不轻松，但它值得做。我们先不追求一步到位，先把关键路径打通。每个人只要把自己负责的部分做好，整体结果就能往前推进。

### 承担责任

> 这次结果没有达到预期，我作为负责人先承担责任。接下来我们不互相甩锅，先把事实复盘清楚：哪里判断错了，哪里协作断了，下一次机制怎么改。

### 授权下属

> 这个方向你来主导，我只给边界：目标是 XX，不能突破的底线是 XX，需要同步的节点是 XX。具体方法你可以自己定，我看结果，也支持你试错。
