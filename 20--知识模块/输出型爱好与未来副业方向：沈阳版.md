# 输出型爱好与未来副业方向：计算机 + 摄影 + 调酒 + FPS 端游

> 目标：选择几个既能长期坚持、又有机会变成副业收入、实习项目和未来职业优势的输出型爱好。  
> 所在地：沈阳，但不把沈阳作为发展边界。  
> 个人情况：计算机专业大一下学生，喜欢调酒、摄影，也喜欢无畏契约、三角洲等 FPS 端游。  
> 核心思路：以计算机专业能力作为长期底盘，用摄影、调酒、端游做差异化场景，先积累作品、项目、内容和接单案例，再决定是副业变现、实习、考研还是就业。

## 先说结论：更适合你的主线

你现在不是一个普通的“想做副业的人”，而是一个还有 3 年大学窗口期的计算机专业学生。这个阶段最值钱的不是马上赚几百块，而是建立一条能同时服务于：

- 接单
- 实习
- 简历
- 作品集
- 未来跳槽
- 个人兴趣表达

的复合路线。

最推荐主线：

**AI 应用开发 / 前端全栈小工具 + 摄影 + 调酒 + FPS 游戏内容工具化**


## 一、兴趣方向总览

下面这些方向都可以做，但它们的优先级不同。对你来说，最重要的是把兴趣变成“项目需求、内容主题和接单入口”，而不是被兴趣消耗时间。

| 方向 | 适合你的用法 | 可能变现路径 | 优先级 |
|---|---|---|---|
| AI 应用开发 / 前端全栈小工具 | 主线技能，做成项目、作品集、接单案例 | 网站接单、AI 小工具、实习、技术岗位 | 最高 |
| 摄影 | 训练审美，提供作品素材，做摄影网站和预约系统 | 写真、活动跟拍、作品集网站打包 | 高 |
| 调酒 | 做内容主题和产品场景，适合开发配方库、酒单工具 | 酒单页、调酒内容、酒吧小工具 | 高 |
| FPS 端游 | 做游戏工具、复盘内容、校园电竞活动 | 工具模板、教学复盘、赛事服务 | 高 |
| 同城内容 / 沈阳素材 | 用作线下拍摄、探店、商家案例来源 | 探店、商家拍摄、本地社群 | 中 |
| AI + 本地商家服务 | 接小单练商业能力 | 文案、海报、菜单页、官网、预约表单 | 中 |
| 手作 / 文创 | 可作为兴趣尝试，不建议作为主线 | 市集、小红书、定制礼物 | 低 |

## 二、最推荐的起步路线

更建议走这条路线：

**计算机技能为主线，摄影、调酒和 FPS 端游作为内容与项目场景。**

原因：

1. 你是计算机专业，大一阶段最应该积累可展示项目，而不是只做零散副业。
2. 摄影、调酒和游戏能让你的项目不无聊，有明确的内容场景和个人标签。
3. AI 应用、前端网站、小程序、数据工具、内容工具，都可以变成接单作品和简历项目。
4. 沈阳可以作为练习和线下素材来源，但不限制你的接单范围和未来就业城市。

路径可以设计为：

计算机基础 -> AI / 前端项目 -> 摄影 / 调酒 / FPS 兴趣项目 -> 作品集 -> 接单 / 实习 / 就业。

## 三、具体方向分析与落地措施

### 1. AI 应用开发 / 前端全栈小工具

这是你的主线技能。它不是单纯“学编程语言”，而是学会把真实需求做成可用产品。

可以做的东西：

- 个人作品集网站
- 摄影预约系统
- 鸡尾酒配方库
- AI 鸡尾酒推荐助手
- FPS 灵敏度换算器
- 游戏训练记录系统
- 校园电竞赛事报名页
- 商家菜单页 / 官网 / 活动页

可落地措施：

1. 先学 HTML / CSS / JavaScript，把页面做出来。
2. 再学 React / Next.js，把项目做成可维护的网站。
3. 学会 Git / GitHub / Vercel，所有项目都要能在线访问。
4. 每个项目都写 README、项目复盘和演示视频。
5. 所有兴趣都尽量项目化，不只停留在“我喜欢”。

变现路径：

- 给个人做作品集网站。
- 给商家做菜单页、活动页、预约表单。
- 给摄影师、社团、校园活动做报名页和展示页。
- 做 AI 小工具或模板售卖。
- 用项目投前端、AI 应用、产品技术实习。

### 2. 摄影 / 短视频剪辑

摄影适合做你的审美标签，也适合给作品集和项目提供素材。

适合练习的方向：

- 人像写真
- 校园活动跟拍
- 调酒静物摄影
- 酒吧 / 咖啡店环境图
- 电竞赛事现场照
- 战队定妆照和选手海报
- 城市街拍和夜景

可落地措施：

1. 先做 10 组样片，不一定收费，重点是积累作品。
2. 每组作品都整理到个人网站里，而不是只发朋友圈。
3. 给每组作品写清楚：拍摄目的、设备、调色思路、适合客户。
4. 尝试“摄影 + 网站”打包服务，比如给同学做个人主页和写真展示页。

变现路径：

- 活动跟拍
- 个人写真
- 社团宣传照
- 电竞赛事摄影
- 摄影作品集网站打包

### 3. 调酒内容 / 鸡尾酒工具

调酒很适合做差异化，因为它天然有配方、场景、审美、摄影和社交属性。

可以输出的内容：

- 鸡尾酒配方
- 家庭调酒入门
- 低预算调酒
- 调酒器具清单
- 酒单设计
- 调酒静物摄影
- 酒吧菜单分析

可落地措施：

1. 每周记录 2-3 款鸡尾酒配方。
2. 每款酒都拍一张照片，写清楚配方、口味、成本、难度。
3. 把这些数据整理成 JSON，后续做成配方库。
4. 再升级成 AI 酒单推荐助手。

变现路径：

- 调酒内容账号
- 酒单设计
- 酒吧 / 咖啡酒馆菜单页
- 鸡尾酒配方库模板
- AI 酒单推荐工具

### 4. FPS 端游内容 / 游戏工具

无畏契约、三角洲这类 FPS 端游可以变现，但不建议主攻纯直播、职业选手或代练。更适合你的方式是：把游戏理解变成内容，把游戏需求变成工具。

适合做的内容：

- 枪法训练计划
- 灵敏度设置
- 地图点位
- 道具教学
- 低分段上分思路
- VOD 复盘
- 三角洲装备收益分析
- 对局复盘和失误总结

适合做的工具：

- FPS 灵敏度换算器
- 无畏契约训练记录系统
- 地图点位查询器
- 三角洲装备收益 / 跑刀成本计算器
- 对局复盘标注工具
- 校园电竞赛事报名与战绩展示页

变现路径：

- 教学复盘
- 游戏工具模板
- 校园电竞赛事服务
- 电竞摄影和短视频剪辑
- 游戏内容账号引流

### 5. 同城内容 / 沈阳线下素材

沈阳不再是你的边界，但可以作为初期素材和获客来源。

可利用场景：

- 中街、西塔、老北市、浑南奥体、铁西工业风街区
- 咖啡店、酒吧、清吧、餐饮店
- 校园活动、社团活动、电竞比赛
- 线下摄影、调酒、观赛局

更适合你的用法：

- 给本地商家做菜单页、官网、预约表单。
- 给校园社团做活动报名页和摄影跟拍。
- 给调酒 / 咖啡 / 餐饮小店做内容和页面案例。
- 用沈阳场景练摄影和短视频，但不把账号定位锁死在沈阳。

### 6. AI + 本地商家服务

这条适合作为小额接单练习，不建议取代开发主线。

可以提供的服务：

- 商家单页官网
- 菜单页
- 活动报名页
- 预约表单
- 小红书标题和文案
- 团购详情页文案
- 海报和活动图
- AI 文案小工具

可落地措施：

1. 先做 2-3 个模拟案例。
2. 找学校周边小店或熟人店铺做低价案例。
3. 交付必须包含线上链接、可修改版本和展示截图。
4. 用案例说话，不要只卖“我会 AI”。

## 四、30 天启动计划

### 第 1 周：搭建个人作品集网站框架

任务：

- 确定个人定位：计算机学生 + 摄影 + 调酒 + FPS 工具。
- 搭建一个简单个人主页。
- 建立 GitHub 仓库。
- 部署到 Vercel / Netlify / Cloudflare Pages。

页面结构：

- 首页：一句话介绍你是谁。
- 项目页：未来放开发项目。
- 摄影页：放精选照片。
- 调酒页：放配方和照片。
- 游戏工具页：预留 FPS 小工具入口。

第 1 周交付物：

- 1 个线上个人主页
- 1 个 GitHub 仓库
- 1 份 README 初稿

### 第 2 周：完成摄影作品页 + 调酒配方页

任务：

- 选 10-20 张照片，整理成摄影作品页。
- 记录 5-10 个鸡尾酒配方。
- 给每个配方写清：基酒、材料、口味、难度、成本、适合场景。
- 给每款酒至少配一张照片或视觉参考。

第 2 周交付物：

- 摄影作品页
- 调酒配方页
- 一份可复用的配方数据结构

### 第 3 周：做一个 FPS 小工具原型

优先做：

**FPS 灵敏度换算器**

功能：

- 输入当前游戏、目标游戏、DPI、当前灵敏度。
- 输出换算后的目标灵敏度。
- 支持无畏契约、三角洲、CS2、Apex 等常见 FPS。
- 给出训练记录备注区。

第 3 周交付物：

- 1 个可访问的 FPS 小工具页面
- 1 份功能说明
- 1 条演示视频或图文介绍

### 第 4 周：发布项目、写复盘、做 3 条内容展示

任务：

- 写一篇项目复盘：《我用 30 天做了一个个人作品集 + FPS 小工具》。
- 做 3 条内容展示：
  - 1 条摄影作品展示
  - 1 条调酒配方展示
  - 1 条 FPS 工具演示
- 把项目链接放到个人简介、简历和作品集里。

第 4 周交付物：

- 1 篇项目复盘
- 3 条内容
- 1 份作品集初版
- 1 份下一阶段计划

## 五、判断是否值得继续的指标

技术指标：

- 是否能独立完成页面、样式、交互和部署。
- 是否能把项目写成 README 和复盘。
- 是否能在下一个项目里复用上一个项目的经验。

内容指标：

- 摄影、调酒、游戏内容是否有人收藏、评论、私信。
- 是否有人问工具链接、配方、拍摄方式或游戏设置。
- 内容是否能反向带来项目灵感。

商业指标：

- 是否有人愿意让你做网站、菜单页、预约页或工具。
- 是否有人愿意付费请你拍照、做赛事图、做活动页。
- 是否能拿到 1-3 个真实案例。

个人指标：

- 你是否愿意持续做 3 个月。
- 输出过程是否让你更兴奋，而不是只觉得消耗。
- 是否积累了作品、代码、案例、内容和复盘。

## 六、我的最终建议

最适合你的组合路线：

1. 技术线：前端开发 / AI 应用开发 / 小工具开发。
2. 内容线：摄影、调酒、FPS 游戏复盘。
3. 项目线：作品集网站、鸡尾酒工具、FPS 工具。
4. 变现线：小网站接单、摄影打包、电竞赛事服务、工具模板。
5. 就业线：前端开发、AI 应用开发、产品技术实习、AIGC 产品 / 运营。

不要一开始就追求“快速副业变现”，先追求“技术基础 + 真实项目 + 可展示作品”。  
当你连续做出 2-3 个线上项目后，再根据反馈选择接单、实习、比赛或内容输出。

## 七、学一门能接单或跳槽的技能

如果只选一门技能，最推荐学习：

**AI 应用开发 / 前端全栈小工具**

这不是单纯“学编程语言”，而是学会把一个真实需求做成可用产品：

- 网页
- 小程序
- 管理后台
- AI 助手
- 数据看板
- 预约系统
- 作品集网站
- 自动化脚本
- 面向商家的轻量工具

### 为什么优先学这门

1. **能接单**  
   很多个人、工作室、小商家需要官网、活动页、预约表单、作品集、菜单页、库存表、自动化表格、AI 文案工具。这类需求不一定大，但适合学生从小单开始。

2. **能跳槽**  
   这门技能对应的岗位包括：前端开发、Web 开发、全栈开发、AI 应用开发、低代码 / 自动化开发、产品技术实习生、数据可视化开发。

3. **和兴趣能打通**  
   摄影、调酒、FPS 游戏都可以成为项目主题，不会让你的作品集千篇一律。

4. **比只做内容更有长期复利**  
   内容账号可能起号失败，但项目作品会沉淀到简历、GitHub、作品集和接单案例里。你大一开始做，到了大三找实习会很占优势。

5. **AI 会放大个人产能**  
   现在写代码、做页面、改样式、生成文案、整理数据、写 README、做演示视频，都可以用 AI 辅助。

### 不建议一开始学什么

不建议一开始只学“算法刷题”或“只看课不做项目”。

原因：

- 算法重要，但短期不能直接接单。
- 只看课容易产生“我学了很多”的错觉。
- 大一最应该尽快进入“学一点，做一点，发出来一点”的循环。
- 没有作品，未来找实习会缺少证明材料。

也不建议把“纯直播、职业电竞、代练”当主线。

原因：

- 纯直播头部效应强，起号不确定性大。
- 职业电竞窗口期短，竞争极强。
- 代练、账号交易、外挂相关风险高，也不适合长期积累信用。

### 推荐技能栈

开发主线：

- HTML / CSS / JavaScript
- TypeScript
- React
- Next.js 或 Vue / Nuxt
- Node.js
- Python
- SQLite / PostgreSQL
- Git / GitHub
- Vercel / Netlify / Cloudflare Pages

AI 工具层：

- OpenAI / 通义千问 / 豆包 / Kimi 等大模型 API
- Prompt 编写
- AI 文案生成
- AI 图片生成
- AI 数据整理
- AI 辅助编程

表达与获客层：

- 摄影
- Lightroom / Photoshop
- 剪映专业版
- 小红书 / B 站 / 抖音
- 作品集网站
- 简历和项目文档

核心能力：

1. 能把一个需求拆成页面、功能和数据结构。
2. 能做出一个可以真实访问的网站或小工具。
3. 能用 AI 加速开发，但自己能看懂和修改代码。
4. 能把摄影、调酒和游戏内容做成项目，而不是只停留在兴趣。
5. 能写清楚项目介绍、技术栈、功能亮点和后续优化。
6. 能把项目放到作品集，用于接单、实习和面试。

### 90 天学习路线

#### 第 1 阶段：0-30 天，学会做一个网页作品

目标：

- 掌握 HTML / CSS / JavaScript 基础。
- 做出第一个摄影作品集网站。
- 学会用 GitHub 保存代码，用 Vercel 或 Netlify 部署。

每天练习：

- 写 30-60 分钟代码。
- 拍 1-3 张照片，作为作品集素材。
- 记录 1 个调酒配方或 1 条游戏训练数据。

第 30 天交付物：

- 1 个个人主页
- 1 个摄影作品集页面
- 1 个调酒配方静态页面
- 1 个 FPS 小工具雏形
- 1 个 GitHub 仓库
- 1 个线上访问链接

#### 第 2 阶段：31-60 天，做出第一个 AI / 数据小工具

目标：

- 做一个和调酒、摄影或 FPS 游戏相关的小工具。
- 项目要有真实功能，而不是只有页面。

项目选题：

1. 鸡尾酒配方搜索器：按基酒、口味、难度筛选。
2. AI 酒单生成器：输入预算、口味、场景，生成酒单。
3. 摄影作品筛选器：按地点、风格、色调管理照片。
4. FPS 灵敏度换算器：按游戏和 DPI 换算灵敏度。
5. 三角洲装备收益计算器：记录投入、撤离、收益和风险。

第 60 天交付物：

- 1 个可访问的 AI / 数据小工具
- 1 篇项目复盘文章
- 1 个演示视频
- 1 份项目 README
- 10 条围绕项目的图文 / 短视频内容

#### 第 3 阶段：61-90 天，开始接小单或准备实习

目标：

- 尝试拿到第一单。
- 或者为大二、大三实习做作品集准备。

接单方向：

- 个人摄影作品集网站
- 小商家官网 / 菜单页
- 活动报名页
- 预约表单
- 校园电竞赛事页
- AI 文案小工具
- 游戏训练工具模板
- 摄影 + 网站打包服务

实习 / 就业方向：

- 前端开发实习
- Web 开发实习
- AI 应用开发实习
- 产品技术实习

### 作品集模板

作品集不要只放截图，要让别人看懂你的技术能力、审美能力和解决问题能力。

每个案例写清楚：

- 服务对象：个人、社团、商家还是自己的兴趣项目
- 目标：展示作品、收集预约、生成推荐、管理数据、提高转化
- 技术栈：React / Next.js / Vue / Node.js / Python / 数据库 / AI API
- 功能亮点：搜索、筛选、表单、登录、数据管理、AI 生成
- 视觉表达：摄影作品、酒单设计、页面风格、交互细节
- 交付内容：线上链接、代码仓库、README、演示视频
- 数据结果：访问量、使用反馈、客户评价；没有数据就写下一步测试计划
- 复盘：下一次怎么优化
