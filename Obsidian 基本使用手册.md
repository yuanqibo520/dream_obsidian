## 🧩 二、嵌入块（Embeds）—— 实现“一处修改，处处更新”

### ✅ 基础
- `![[笔记名]]`：嵌入整篇笔记
- `![[笔记名#^块ID]]`：嵌入特定段落（需先给段落加 `^块ID`）

### 🔥 高级玩法
#### 1. **制作“动态知识卡片”**
- 在 `[[四级备考]]` 中嵌入：
  ```markdown
  ## 最新计划
  ![[2025四级倒计时]]
  
  ## 高频词汇
  ![[四级核心词表#^week10]]
  ```
- 修改 `2025四级倒计时.md`，所有嵌入处自动更新

#### 2. **构建“仪表盘”笔记**
- 创建 `[[我的今日看板]]`，嵌入：
  - 今日任务（来自 Daily Note）
  - 项目进度（`![[四级备考#进度]]`）
  - 健康记录（`![[肌酸服用日志]]`）
  → 打开一个笔记，掌控全局

#### 3. **跨领域整合**
- 在 `[[沈阳航大生活]]` 中嵌入：
  ```markdown
  ## 交通指南
  ![[沈阳航大地铁指引]]
  
  ## 校园卡服务
  ![[校园卡办理流程]]
  ```
- 避免重复复制，保证信息一致性

## 🧪 三、配合 Git —— 实现版本控制 + 多端同步 + 安全备份

### ✅ 基础
- Git 记录每次修改，可回退到任意历史版本

### 🔥 高级玩法
#### 1. **自动提交脚本（推荐）**
- 安装插件 **“Obsidian Git”**（社区插件）
- 设置 **每 30 分钟自动 push 到 GitHub/Gitee**
  → 即使电脑损坏，知识库毫发无损

#### 2. **忽略敏感/临时文件**
- 在库根目录创建 `.gitignore` 文件，内容：
  ```gitignore
  # 忽略 Obsidian 设置（避免多设备冲突）
  .obsidian/workspace
  .obsidian/plugins/
  
  # 忽略临时文件
  *.tmp
  ```
  → 只同步**笔记内容**，不同步界面布局

#### 3. **用 Git 分支做“实验性整理”**
- 主分支 `main`：稳定知识库
- 新建分支 `experiment/restructure`：尝试重组文件夹
- 整理满意后 `merge`，不满意可丢弃，**主库不受影响**

#### 4. **与他人协作（可选）**
- 把库设为私有 GitHub 仓库
- 家人/导师可提 Issue 讨论内容（如“四级计划是否合理？”）

## 🧠 组合技：三者联动工作流

> **场景**：你想优化“摄影知识体系”

1. 在 `[[摄影]]` MOC 中，用双向链接列出所有子主题  
2. 在 `[[相机电影模式参数]]` 中，嵌入 `![[曝光参数表#^电影模式]]`  
3. 修改参数后，执行 `Git commit -m "更新电影模式参数"`  
4. 所有引用该参数的地方自动更新 + 历史可追溯

---

## ⚠️ 注意事项
- 嵌入块**不能无限嵌套**（避免循环引用）
- Git 同步前，**关闭 Obsidian**（防止文件锁冲突）
- MOC 笔记**不要嵌入自身**

截至 2025 年，Obsidian 社区中最受欢迎、下载量最多的插件通常包括以下这些（按功能分类，非严格排名）：

1. **Templater** – 高级模板系统，支持动态内容（如日期、变量、脚本）。
2. **Dataview** – 用类 SQL 语法查询和展示笔记中的数据，实现数据库式管理。
3. **Quick Add** – 快速创建笔记、插入内容或执行自定义操作。
4. **Obsidian Git** – 自动或手动将笔记库同步到 Git 仓库，便于版本控制与备份。
5. **Advanced Tables** – 增强表格编辑功能，支持排序、对齐、快捷键等。
6. **Tag Wrangler** – 管理标签，支持重命名、合并、自动补全。
7. **Calendar** – 日历视图，方便按日期查看和创建日记类笔记。
8. **Kanban** – 创建看板（如 To-Do、进行中、完成），适合任务与项目管理。
9. **Sliding Panes (aka Hover Editor)** – 增强多窗格浏览体验，支持悬停预览与灵活布局。
10. **Note Refactor** – 批量重命名、移动、重构笔记及其链接。

这些插件在 Obsidian 官方社区插件市场中长期位居前列，具体排名可能因版本更新与用户偏好略有变化。建议在 Obsidian 的「社区插件」浏览器中按「Downloads」排序查看最新情况。
