## 主要笔记类型与使用建议

- **MOC（内容地图/主题索引）**：在「00--index」目录下维护各主题索引页（如“摄影”“学习路线”），通过双向链接整合子主题内容，便于快速跳转与仪表盘搭建，构建清晰的知识脉络。

- **模板（Templates）**：将常用模板集中存放于「40--资源库/笔记模板/」，新建笔记时可直接复制复用，或通过 Obsidian 模板插件快速调用，确保笔记结构统一、内容可复用。

- **项目与任务（Projects & Tasks）**：按“长期项目+短期任务”分类管理，为每个项目搭建专属 MOC，嵌入进度跟踪、日程规划与行动项，建议借助数据库或 YAML 元数据记录任务状态与截止时间，提升项目管控效率。

- **日常笔记 / Daily Notes**：建议将“今日计划/今日看板”纳入仪表盘，通过嵌入语法 `![[Daily Note]]` 及查询块（Dataview/Tasks 插件）汇总当日任务与进度，实现日常工作的可视化管理。

- **杂记与零碎想法**：临时灵感或碎片化内容可先记录至「00--index/杂记.md」，定期开展“整理入库”工作，将有价值的内容迁移至对应 MOC 或优化为结构化笔记，避免知识碎片化。

## 工作流建议（收集 → 处理 → 组织 → 复盘）

1. **收集（Capture）**：将想法、待办、资料随手记录至每日笔记或临时笔记，优先保证记录速度，无需提前梳理结构。

2. **处理（Process）**：每日或每周对临时笔记进行分类筛选，决定内容的处置方式——丢弃无效信息、归档有用内容、转化为待办任务，或拆解为结构化笔记。

3. **组织（Organize）**：将成熟内容与对应 MOC、模板关联，通过链接与嵌入功能保持“单一事实源”（single source of truth），确保知识的一致性与可追溯性。

4. **复盘（Review）**：定期复盘项目进度与笔记质量，优化模板设计与 MOC 结构，持续提升知识库的实用性与可检索性。

## 同步与版本控制（Git）

推荐通过 Git 管理笔记版本，实现云端同步与历史追溯，具体建议如下：

- **同步原则**：依托 Git 实现本地与云端/私人仓库的同步，留存完整的笔记修改记录。

- **注意事项**：
       

    - 执行 Git 提交/推送操作前，务必关闭 Obsidian 客户端，避免文件锁定或未保存状态导致的同步冲突，此要点已在《Obsidian 基本使用手册》中明确说明。

    - 提交信息需清晰规范，注明笔记主题与核心改动，示例：`git commit -m "更新：四级备考#词表"`。

- **基本命令示例**：
        `# 拉取远程最新版本（多设备同步前执行）
git pull origin main

# 提交本地修改并推送至远程
git add . && git commit -m "更新笔记" && git push origin main`

## 常见问题与注意事项

- 嵌入块（Embeds）不可无限嵌套，同时需避免循环引用，防止 Obsidian 客户端卡顿或显示异常。

- MOC 笔记禁止嵌入自身，避免造成知识关联混乱与展示错误。

- 严格遵循“关闭 Obsidian 后同步 Git”的原则，最大限度减少文件冲突与数据损坏风险。

- 定期开展“笔记健康检查”，排查死链接、未分类笔记与重复条目，保持知识库的整洁性与可用性。
